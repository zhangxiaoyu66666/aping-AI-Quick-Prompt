# 啊拼功能省时性审计

本文检查啊拼当前功能是否真的能让对应用户少走步骤，而不是只把“打开大模型官网写提示词”换成“在本应用里写提示词”。

审计范围：GitHub 社区版当前工作区。Microsoft Store 独占功能和未发布实验功能不纳入本文。

参考入口：

- `README.zh-CN.md`
- `src/PromptInputMethod.App/CompactPromptWindow.xaml`
- `src/PromptInputMethod.App/CompactPromptWindow.xaml.cs`
- `src/PromptInputMethod.App/Services/OptimizationTargetService.cs`
- `src/PromptInputMethod.App/Services/PromptTemplateCatalogService.cs`

## 判断标准

一个功能只有在移除真实重复动作时，才算有效省时：

- 少开一个网站或软件。
- 少一次复制、翻译、再粘贴。
- 少一次手动拆字段。
- 少一次从模板库里找半天。
- 少一次重复写提示词约束。
- 少一次把截图文字手打进模型。
- 少一次解释“你要按某个 Skill / 工作流做”。

如果功能只是让界面更完整，但用户仍然要手动完成关键步骤，则只能算半省时。

## 总体结论

啊拼目前最明确的省时价值在三类用户：

1. 中文用户使用英文偏好的模型或生成工具：双语输出和一次性英文同步可以省掉翻译网站或第二轮模型翻译。
2. AIGC 创作者：文生图、ComfyUI / Stable Diffusion、即梦 / Seedance、女性人像 Skill 能减少提示词结构化时间。
3. 经常复用提示词的用户：常用提示词、历史记录、模板、Skill 挂载能减少重复构造提示词。

但是目前还没有完全打穿所有目标平台。很多功能已经能省掉“去大模型官网生成提示词”的步骤，但还没有省掉“把结果自动送进目标平台”的步骤。当前已补上字段级复制：ComfyUI / Stable Diffusion 可复制正向、反向、参数；即梦 / Seedance 可复制分镜、约束、参数；Veo 可复制镜头、约束、时长；AI 编程可复制任务、约束、验证；论文去 AI 味可复制原文、禁用约束、输出规则。

|---|---|---|---|---|---|---|
| ComfyUI / Stable Diffusion 适配 | B 站 UP、AIGC 创作者、SD WebUI 用户 | 打开大模型官网，中文写需求，复制结果，翻译英文，再拆成 positive / negative / 参数，粘到 ComfyUI | 中文输入后直接得到中文主输出和英文可执行输出；新增正向、反向、参数一键复制 | 明显省时 | 仍需手动粘贴到 ComfyUI；不能读取用户 workflow JSON 并自动 patch 节点 | 下一步支持导入 workflow JSON，按节点标题或 ID 输出 patch；可选接入本地 ComfyUI API |

| Veo 3 优化 | 英文视频生成用户、广告分镜用户 | 把中文需求翻译成英文导演提示词，再补镜头、运动、声音、比例 | 直接生成英文电影导演式结构，避免自己组织镜头语言；可一键复制镜头、约束、时长比例字段 | 有省时 | 缺少平台 presets 和官方 API 投递 | 增加时长、比例、镜头风格 presets；保留手动粘贴为默认 |
| 女性人像提示词导演 Skill | 人像写真、电商试衣、服装搭配、参考图保留用户 | 找 Skill 文档或模板，复制到模型，再粘需求，让模型套规则 | 内置 Skill 直接作为模板源和命中上下文，减少查文档和复制 Skill 的动作 | 明显省时 | 输出结果仍未按人像工作流拆成“风格、服装、姿态、负面、参数”快速复制 | 增加人像专用字段复制；按写真、电商、古风、生活方式等场景给快捷 presets |
| 文生图通用目标 | 轻量图像创作者 | 自己想主体、风格、镜头、负面词和比例 | 本地结构草稿能快速补齐主体、风格、光线、构图、负面约束 | 有省时 | 不同平台对提示词格式差异很大，通用输出容易变成模板腔 | 推荐用户优先选具体平台目标，而不是停留在通用文生图 |
| 双语输出 | 中文用户使用英文工具、跨平台创作者 | 中文需求生成后再去 Google 翻译，或让模型第二轮翻译 | 同一轮模型调用内同时生成中文主提示词和英文提示词 | 明显省时 | 如果模型不按协议输出英文，只能本地镜像兜底，质量不如模型原生英文 | 继续强化每个目标的英文结构规则；对英文缺失给明显提示 |
| Markdown 去除 | 需要直接粘贴 prompt 的用户 | 模型输出 Markdown 标题、列表、代码块，用户手动清理 | 输出框自动清理 Markdown 包装，减少复制前整理 | 小幅省时 | 过度清理可能误删用户确实想保留的符号 | 保留设置开关；对代码类提示词不要误删必要格式 |
| OCR 图片 / OCR 截图 | 需要从网页、PDF、游戏、截图里取文字的用户 | 截图，打开 OCR 工具，复制文字，再贴到模型 | 在啊拼内直接 OCR 图片或截图，作为可编辑需求或上下文 | 明显省时 | OCR 只解决文字搬运，不理解截图布局；截图等待时机需要避免动画残影 | 后续加入“识别后作为背景上下文 / 直接加入需求”的明确模式 |
| 图片直发大模型 | 多模态模型用户、参考图提示词用户 | 保存图片，打开模型官网，上传图片，再写需求 | 在输入区直接附图或截图并发送给支持图片的模型 | 有省时 | 缺少 provider 多模态能力提示；不支持的模型可能让用户困惑 | 模型管理里标注是否支持图片输入，并在发送前提醒 |
| 聊天式需求优化 | 不会一次写完整需求的普通用户 | 每轮重新组织上下文，模型容易忘记前面补充 | 用户可以连续补充，程序把上一版提示词和本轮补充合并 | 有明显省时 | 如果目标切换或新会话提示不清，可能把旧目标带入新任务 | 保持“新会话切换目标”的提示；后续可显示当前锁定目标 |
| 深度思考与等待气泡 | 使用 reasoning 模型的用户 | 发出请求后不知道模型是卡死还是在思考 | 显示正在思考和已思考时间，流式接收减少等待焦虑 | 辅助省时 | 它不缩短模型实际推理时间，只减少误判和重复点击 | 保持真实计时；避免把思考摘要当成最终提示词 |
| 常用提示词 / 收藏 | 高频复用模板的个人用户 | 每次从历史或文档里找同一段提示词 | 一键收藏、快速复制、编辑、删除 | 明显省时 | 如果没有分类、搜索或排序，收藏多了会反过来拖慢 | 保持搜索和分页；增加最近使用排序 |
| 历史记录 | 经常复用旧需求的用户 | 翻聊天记录或模型官网历史，复制旧 prompt | 本地历史可回填旧需求、中文提示词和英文提示词 | 有明显省时 | 多设备或多版本历史暂不在 GitHub 社区版同步 | 保持本地优先；导出 JSONL / 快照用于手动备份 |
| 模板库 | 写作、翻译、开发、图片提示词用户 | 去各个 prompt 仓库、网站、收藏夹找模板 | 内置模板源可搜索、下拉选择，直接插入当前需求 | 有省时 | 模板太多会造成选择成本；模板如果不按目标过滤会拖慢 | 继续按优化目标过滤模板；常用模板应提升到顶部 |
| Skill 挂载 | 有固定工作流的高级用户、Agent 用户 | 每次把 `SKILL.md` 内容复制进模型上下文 | 挂载一次后自动保存、提取触发词、命中时注入上下文 | 明显省时 | 目前是文本工作流，不执行外部脚本；用户可能期待真正插件能力 | UI 上保持“Skill 是文本工作流”的边界；后续可支持安全脚本但需确认 |
| AI 编程目标 | 开发者、使用 Codex / Claude Code / Cursor 的用户 | 临时写“先读代码、最小修改、跑测试”等约束 | 内置根因分析、最小修复、UI 修改、代码审查、新增功能等模板；可一键复制任务、约束、验证字段 | 有省时 | 仍需用户自己把仓库信息、错误日志、文件路径贴进来 | 增加“复制命令输出模板”和错误日志整理 |
| Skill 体系模板 | 想写 Codex / Claude Skill 的用户 | 从零组织 Skill 目录、触发条件、工作流、安全边界 | 直接生成 Skill 设计框架和检查清单 | 有省时 | 只生成文本，不自动创建文件夹或校验 Skill 结构 | 可增加“导出 Skill 文件夹”功能 |
| 论文去 AI 味 | 学生、论文润色用户 | 临时写改写要求，反复提醒不要编造、不要模板腔 | 内置自然学术改写规则和禁用词约束；可一键复制原文、禁用约束、输出规则字段 | 有省时 | 仍需用户自己粘贴原文；不支持 Word / PDF 直接导入 | 增加文档导入和段落级批处理时才会更像完整工具 |
| 模型管理与 Provider 验证 | 多模型用户、本地模型用户 | 手动记 base URL、模型名、API key，失败时自己判断 | 应用内保存 provider、模型名、Credential Manager 密钥，并验证端点；显示 provider、深度思考、流式、图片输入、语言适配标签 | 有省时 | 图片输入能力是保守识别，不等于官方确认 | 后续维护 provider 能力矩阵和失败原因分类 |
| 本地敏感信息脱敏 | 会把日志、代码、截图发给模型的用户 | 手动删 API key、token、隐私字段 | 发送前做本地红线过滤，减少手动检查压力 | 辅助省时 | 不能当正式 DLP；误报和漏报都可能存在 | UI 明确“减少风险，不保证完全安全”；允许用户查看脱敏结果 |
| 全局快捷键 / 托盘 / 关闭不退出 | 高频使用者 | 在任务栏找应用或重新启动 | 快捷键和托盘可以随时呼出，减少上下文切换 | 明显省时 | 托盘退出必须真正结束所有进程，否则用户会觉得失控 | 保持“退出即结束进程”的实现；托盘菜单继续简洁 |
| 紧凑窗口 | 写作、开发、生成工具旁边工作的人 | 主窗口占屏，需要来回切窗口 | 小窗可贴边使用，和目标软件并排 | 有明显省时 | 小窗里如果看不到提示词或缺少 Skill / 目标选择，就会变成残缺版 | 保持大小窗同一套输入、目标、模板、Skill、输出逻辑 |
| UI 本地化 / 语言包 | 中英文用户、开源贡献者 | 手动忍受半英半中界面或改源码 | 使用 WinUI 官方资源体系和语言包挂载 | 辅助省时 | 文案覆盖不完整时会破坏专业感 | 新增 UI 文案必须进资源，不要散落硬编码 |
| 导入 / 导出 JSON | 模板维护者、团队共享模板用户 | 手动复制多条模板或常用提示词 | 批量迁移模板、常用提示词、语言包 | 有省时 | 对普通用户价值不高；缺少冲突合并说明时可能覆盖 | 增加导入预览、冲突检测、来源标签 |

## 用户画像视角

### B 站 UP / AIGC 创作者

最省时功能：

- ComfyUI / Stable Diffusion 字段适配。
- 正向、反向、参数一键复制。
- 即梦 / Seedance 导演目标。
- OCR 截图读取参考图文字。
- 女性人像提示词导演 Skill。
- 常用提示词收藏。

还没打穿的地方：

- 不能直接写入 ComfyUI workflow。
- 不能自动识别用户当前 ComfyUI 节点名。
- 即梦 / Veo 已有字段复制；仍不能自动投递到平台。
- 平台参数 presets 还不够产品化。

### 开发者 / 代码智能体用户

最省时功能：

- AI 编程模板。
- Skill 体系模板。
- 全局快捷键和小窗。
- 历史记录和常用提示词。
- 模型管理。

还没打穿的地方：

- 不能自动收集仓库上下文、错误日志、diff。
- 不能一键把输出投递给 Codex / Claude Code / Cursor。
- 代码类输出和 Markdown 去除之间需要更精细边界。

### 学生 / 写作者 / 论文用户

最省时功能：

- 论文去 AI 味目标。
- OCR 图片 / 截图。
- 中文主输出和英文输出。
- 常用提示词与历史。

还没打穿的地方：

- 不支持 Word / PDF 段落批量导入。
- 不支持引用、术语、格式要求的专门收集。
- 输出后还需要手动粘回文档。

### Prompt 模板维护者

最省时功能：

- 模板库。
- JSON 导入导出。
- Skill 挂载。
- 本地化语言包。

还没打穿的地方：

- 模板质量评估和去重还不够自动化。
- 导入冲突提示不够强。
- 没有模板使用频次统计。

## 优先级建议

### P0：把半自动目标都拆成字段复制（已落地）

当前已按这个原则落地：

- ComfyUI / Stable Diffusion：复制正向、反向、参数。
- 即梦 / Seedance：复制分镜、负面约束、平台参数。
- Veo：复制镜头、连续性约束、时长与比例。
- AI 编程：复制任务描述、执行约束、验证字段。
- 论文：复制待处理文本、禁用约束、输出规则。

目标是让用户不用从一整段里手动抠。

### P1：让优化目标从“用户选择”变成“系统建议”（已落地）

用户仍停留在 `通用 LLM` 时，输入“来个美图”“画一张海报”“帮我修 bug”“论文降 AI 味”等明显需求，应用会在生成前自动切到更合适的目标：

- 文生图 -> ComfyUI / Stable Diffusion 或通用文生图。
- 短视频 -> 即梦 / Seedance 或 Veo。
- 代码错误 -> AI 编程根因分析或最小修复。
- 论文文本 -> 论文去 AI 味。

这能省掉用户理解功能分类的时间。为了避免误切，用户已经手动选择目标或当前会话已经锁定目标时不会自动覆盖。

### P1：目标平台能力标签（已落地）

模型和目标平台已显示能力标签：

- 是否支持图片输入。
- 是否支持 reasoning。
- 是否支持流式。
- 是否适合英文提示词。
- 是否适合中文提示词。

这能减少“为什么调用失败”“为什么图片没生效”的排查时间。当前图片输入标签是保守启发式识别，不能替代 provider 官方文档。

### P2：从文本工作台升级到轻量交付器

长期看，最省时间的不是生成漂亮提示词，而是把提示词送到正确位置：

- ComfyUI：workflow JSON patch 或本地 API。
- Stable Diffusion WebUI：参数包复制或 API。
- 即梦 / Veo：平台字段包和剪贴板批量复制。
- Codex / Claude Code：任务包、约束包、验证命令包。

但这类功能涉及平台 API、隐私、授权和稳定性，应该逐个目标做，不要一次性铺开。

## 保留和砍功能的原则

建议以后每个新功能都先回答三句话：

1. 它替用户少做了哪一步？
2. 它是否减少了跨软件、跨网站、跨语言、跨格式的切换？
3. 如果用户仍然要手动拆字段、翻译、复制多次，那下一步怎么补齐？

回答不了这三句的功能，不急着加入口；否则会变成“功能很多，但用户还是慢”。
