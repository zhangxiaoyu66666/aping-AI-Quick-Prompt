## OCR Release Decision

Fire Eye OCR may be included in AI Quick Prompt 1.0 release binaries. The embedded model assets are reviewed as PaddleOCR / PP-OCRv5 Apache-2.0 assets, with converted `.mnn` and fp16 artifacts treated as modified Apache-2.0 deployment artifacts. Keep the Apache-2.0 notices and the hashes in `docs/ocr-model-license-review.md`.

The raw `assets/fire_eye/` folder remains ignored by Git because it is a large local asset cache. That does not block release binaries from shipping `fire-eye-ocr-worker.exe`.

## Required Commands

Run non-GUI release checks:

```powershell
dotnet run --project tests\PromptInputMethod.ReleaseChecks\PromptInputMethod.ReleaseChecks.csproj --configuration Release
```

Run WinUI build:

```powershell
& "C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\amd64\MSBuild.exe" src\PromptInputMethod.App\PromptInputMethod.App.csproj /t:Build /p:Configuration=Debug /p:Platform=x64 /p:OutputPath=bin\CodexVerify\ /m /v:minimal
```

Capture UI regression screenshots:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\Invoke-UiScreenshotChecks.ps1
```

Build the public demo package:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\New-PublicDemoPackage.ps1
```
