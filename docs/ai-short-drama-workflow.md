# AI 短剧工作流研究与啊拼落地方案

日期：2026-06-06

本文目标不是再写一份泛泛的文生视频行业报告，而是把行业里的短剧生产方法拆成啊拼可以承接的产品工作流。结论先写在前面：

> 啊拼要做的不是“一个更长的视频提示词模板”，而是一个短剧导演工作流：一句话生成剧情和角色，角色沉淀为资产，资产生成三视图，剧情拆成 5-10 秒镜头卡，每段镜头生成后再把结尾状态带入下一段。

啊拼当前不直接调用即梦、Seedance、Veo、Runway、Sora、可灵等平台 API，不负责渲染、剪辑、发布或备案。啊拼最适合的位置是 AI 短剧的“前置导演台”和“连续性管理层”：把用户的想法变成可复制、可追踪、可回退、可继续的资产与镜头提示词。

## 1. 行业工作流观察

### 1.1 传统微短剧是“制、宣、发”一体化

微短剧行业不是单点创作，而是从立项、拍摄制作、剪辑播出到投流反馈的链路。艾瑞咨询的行业报告把典型机构流程概括为立项准备、拍摄制作、剪辑播出，并强调需求把控、剧本打磨、制作小组、中台协同、边剪边播、根据审核和数据反馈调整。报告也指出，AIGC 正在作用于故事创意、人物小传、对话脚本、分镜设计、视觉素材、配音音乐和运营反馈等环节。

对啊拼的启发：

- 不要只做“生成一个视频提示词”。
- 要把短剧拆成可复用资产：剧情种子、角色、场景、镜头、声音、字幕、连续性状态。
- 要允许快速迭代和回退：上一段结果不好，改需求后重新生成；上一段满意，就继续下一段。

参考来源：[艾瑞咨询《2024年中国微短剧行业研究报告》](https://pdf.dfcfw.com/pdf/H3_AP202407071637643449_1.pdf)

### 1.2 AI 视频平台普遍鼓励短片段、清晰动作、镜头语言

Runway Gen-4 官方文档说明，Gen-4 基于输入图片和文本提示生成 5 或 10 秒视频，并强调提示词应描述画面中实际发生的动作，而不是抽象概念或否定式要求。它还建议一次只增加一个新元素，便于判断哪些修改改善了结果。

Google Veo 的官方提示词指南把视频提示拆成主体、动作、场景/上下文、时间元素、音频等关键组件，并强调清晰拆解比笼统描述更有效。Veo 文档还把音效、环境声、对白列为可指导视频情绪和节奏的提示元素。

OpenAI Sora 帮助文档也给出同样趋势：从核心意图开始，逐步添加镜头、动作、节奏；可以用 storyboard 逐秒规划，用 stitch / reorder / extend / reprompt / remix 把多个片段连接成长故事。

对啊拼的启发：

- 单段镜头卡默认应控制在 5-10 秒。
- 每段都要写清：主体、动作、场景、镜头、时间轴、声音、结尾画面。
- 生成下一段时，不是“重新生成剧情”，而是从上一段的最后一帧继续。
- 失败修正应是局部迭代：改镜头、改动作、改角色锁定，而不是每次推翻全部提示词。

参考来源：

- [Runway Gen-4 Video Prompting Guide](https://help.runwayml.com/hc/en-us/articles/39789879462419-Gen-4-Video-Prompting-Guide)
- [Runway Gen-4 References](https://academy.runwayml.com/tutorial/gen-4-references)
- [Google Veo video generation prompt guide](https://docs.cloud.google.com/vertex-ai/generative-ai/docs/video/video-gen-prompt-guide?hl=en)
- [OpenAI Help Center: Creating videos with Sora](https://help.openai.com/en/articles/12460853)

### 1.3 角色一致性是短剧工作流的核心，不是锦上添花

短剧和普通风景视频不同，观众追的是角色和冲突。AI 视频一旦脸变了、衣服换了、道具消失了、年龄感漂移了，剧情可信度就断掉。行业里通常用角色设定、参考图、角色卡、首帧/尾帧和多段剪辑来维持一致性。

对啊拼的启发：

- 三视图不应该是第一步。第一步应该是“一句话剧情与角色”，先知道角色在剧情里的功能，再生成三视图。
- 三视图不是单纯出图，而是角色资产卡的一部分：正面、侧面、背面、表情、手势、服装、道具、禁止变化项。
- 每段镜头都必须引用角色资产卡，不要让镜头提示词自己临时创造角色。

### 1.4 合规与发布不是后置细节

国家广播电视总局关于网络微短剧的通知强调许可准入、备案管理、内容审核、行业自律，并要求微短剧上线传播前标注许可证号或备案号。对啊拼来说，虽然当前不负责发布，但工作流里必须留出“合规检查”和“发布资料准备”的位置。

对啊拼的启发：

- 啊拼不应承诺“自动发布短剧”。
- 面向国内公开传播时，文档和 UI 应提示用户关注备案、版权、肖像、平台审核、AI 生成标识、未成年人保护等问题。
- 角色资产和剧情草稿可能是敏感创意资产，默认应本地保存，图片外发必须尊重隐私设置。

参考来源：[国家广播电视总局办公厅关于进一步统筹发展和安全促进网络微短剧行业健康繁荣发展的通知](https://www.nrta.gov.cn/art/2025/2/5/art_113_70148.html)

## 2. 适合啊拼的短剧工作流

啊拼短剧工作流建议定义为 8 个阶段。

### 2.1 一句话企划

输入可以非常短，例如：

> 被退婚的女主回到宴会复仇。

啊拼需要生成：

- 短剧 Logline：主角、欲望、阻碍、爽点或悬念。
- 类型定位：复仇、甜宠、悬疑、逆袭、都市、古风、科幻、Galgame 分支演出等。
- 核心冲突：主角想要什么，谁阻止，为什么此刻爆发。
- 主角功能：欲望、弱点、秘密、情绪弧线。
- 对手/阻力功能：反派、误会、制度、家庭、时间压力或内心阻碍。
- 第一段钩子：第一个 5-10 秒必须出现的画面或台词。

产物名称：`StorySeed`

### 2.2 角色资产卡

基于剧情功能生成角色，而不是先随便生成漂亮人设。

每个角色资产卡包含：

- 角色名
- 剧情功能
- 身份和社会位置
- 性格与欲望
- 弱点或秘密
- 外貌锚点
- 服装锚点
- 道具锚点
- 表情库
- 声音/台词风格
- 禁止变化项

产物名称：`CharacterAsset`

### 2.3 三视图与表情参考

三视图用于指挥画图 AI 生成角色参考，不直接等同于视频镜头。

每个角色应生成：

- 正面全身设定图提示词
- 侧面全身设定图提示词
- 背面全身设定图提示词
- 3-5 个表情或手势参考
- 角色一致性锁定字段
- 负面约束

产物名称：`TurnaroundPrompt`

### 2.4 场景与道具资产

短剧不是只有人物，场景和道具也要锁定。

每个场景资产卡包含：

- 场景名
- 时间和地点
- 空间结构
- 光线方向
- 色彩和质感
- 关键道具
- 可变化项
- 禁止变化项
- 适用镜头段落

产物名称：`SceneAsset`

### 2.5 剧情拆段

行业短剧讲究强钩子和持续推进。AI 视频生成又天然适合短片段，因此啊拼不应该先生成一整集视频提示词，而应该拆成镜头段。

每个段落建议 5-10 秒，结构为：

- 段落编号
- 本段剧情目的
- 开场画面
- 核心动作
- 情绪变化
- 台词/字幕
- 结尾画面
- 下一段钩子

产物名称：`StoryBeat`

### 2.6 镜头卡

镜头卡是啊拼短剧工作流的核心。它面向 AI 视频平台，必须足够具体。

每张镜头卡包含：

- 时长：5-10 秒
- 画幅：9:16 / 16:9 / 1:1
- 平台：即梦 / Seedance / Veo / Runway / Sora / 可灵 / 自定义
- 角色资产引用
- 场景资产引用
- 开场画面
- 主体与场景
- 景别
- 视角
- 镜头焦段感
- 运镜：推、拉、摇、移、环绕、跟拍、手持、稳定器、拉焦等
- 表演调度：眼神、微表情、停顿、呼吸、手部动作、距离变化
- 时间轴：0-2s、2-5s、5-8s、8-10s
- 台词/字幕
- 声音：环境声、BGM、音效、对白语气
- 视觉风格
- 负面约束
- 结尾状态

产物名称：`ShotCard`

### 2.7 生成尝试与失败诊断

每次把镜头卡复制到平台生成后，都应该记录一次尝试。

记录内容：

- 使用平台
- 使用提示词
- 参考图/首帧/尾帧
- 输出结果路径或手动备注
- 成功点
- 失败点
- 失败标签：角色漂移、服装漂移、场景跳变、镜头乱动、口型错位、字幕乱码、动作过大、节奏不清、画面低质等
- 下一次修改建议

产物名称：`GenerationAttempt`

### 2.8 继续下一段

继续下一段时，啊拼必须带入上一段的结尾状态，而不是只带入整段提示词。

必须继承：

- 上一段最后一帧
- 人物站位
- 表情和情绪状态
- 场景和光线方向
- 镜头状态
- 已发生剧情
- 角色资产锁定
- 下一段剧情目的

产物名称：`ContinuityState`

## 3. 啊拼当前已经具备的基础

当前项目已经有一部分能力可以直接支撑短剧工作台：

- 优化目标系统：可以新增 `AI短剧导演工作台`，并通过目标规则约束输出结构。
- 模板源系统：可以挂载 `短剧工作台` 模板源，提供 AI 带做、新手开做、剧本、分镜、角色三视图、5-10 秒运镜、继续下一段、质检返工和交付包模板。
- 聊天式连续优化：用户可以补充角色、台词、画幅、风格、失败反馈。
- 回退上一轮：可以退回上一轮成果并修改上一轮用户需求，适合短剧段落反复调整。
- 中文 + 英文输出：中文保留剧情和台词，英文可服务 Veo、Runway、Sora 等平台。
- 字段复制按钮：短剧目标下可以复制剧本、分镜、镜头字段。
- 历史记录：可保存每次提示词结果，但还不是项目级资产库。
- 图片/截图/OCR：可导入参考图、首尾帧、台词图、场景资料。
- 隐私设置：图片外发需要尊重用户设置，适合处理未公开角色图和剧情草稿。

## 4. 当前缺口

### 4.1 缺项目级状态

现在的历史记录更像“单次提示词记录”，短剧需要“项目”。

需要新增：

- `ShortDramaProject`
- `CharacterAsset`
- `SceneAsset`
- `StoryBeat`
- `ShotCard`
- `GenerationAttempt`
- `ContinuityState`

### 4.2 缺可视化时间线

现在只能在文本框里看提示词。短剧需要段落列表：

- 第 1 段
- 第 2 段
- 第 3 段
- 每段状态：草稿、已生成、已采用、需重试、已导出

### 4.3 缺资产引用

现在模板可以写角色锚点，但应用没有真正的角色资产库。应支持：

- 角色卡列表
- 三视图提示词
- 参考图路径
- 禁止变化项
- 在镜头卡中选择角色资产

### 4.4 缺失败诊断闭环

行业工作流强调快速迭代。啊拼应让用户选择失败原因，然后自动改写镜头卡。

例如：

- 脸变了 -> 强化角色锚点、参考图作用、减少多人同框。
- 镜头乱动 -> 改成锁定镜头或单一运镜。
- 动作太大 -> 降低动作幅度，拆成两段。
- 台词乱 -> 缩短对白，分角色写清。
- 字幕乱码 -> 改为后期字幕，不让视频模型生成文字。

## 6. UI 落地建议

### 6.1 第一版：不做大改页面，先在现有工作台中完成闭环

当前最小可用版本：

- 新增 `AI短剧导演工作台` 优化目标。
- 新增 `短剧工作台` 模板源。
- 新增模板：
  - AI 手把手带做
  - 新手从 0 开做
  - 想法生成剧本
  - 剧本拆分镜
  - 一句话生成剧情角色
  - 角色三视图资产卡
  - 5-10 秒大师运镜
  - 继续下一段
  - 质检返工
  - 整理制作交付包
- 字段复制按钮在短剧目标下显示：
  - 剧本
  - 分镜
  - 镜头
- 如果已有结果，输入框为空时点击发送，默认根据 `AI带做进度` 推进下一步：想法后做剧本，剧本后拆分镜，分镜后做三视图或运镜，发现缺口先质检返工，镜头后继续下一段，内容完整后整理交付包。
- 利用已有“退回上一轮”修改上一轮需求。

这已经能完成一个文字型工作流。

### 6.2 第二版：短剧项目抽屉

在右侧或独立页面新增项目抽屉：

- 项目信息
- 角色资产
- 场景资产
- 镜头列表
- 生成尝试记录

### 6.3 第三版：镜头时间线

新增时间线视图：

- 每个镜头一张卡。
- 点击镜头卡可编辑开场、结尾、运镜、台词、声音。
- 支持复制当前镜头提示词。
- 支持从当前镜头继续下一段。
- 支持标记“采用 / 重试 / 废弃”。

### 6.4 第四版：失败诊断

在每个镜头结果下增加失败标签：

- 角色漂移
- 服装变化
- 场景跳变
- 镜头乱动
- 口型错位
- 字幕乱码
- 动作过猛
- 节奏不清
- 风格跑偏

用户点失败标签后，啊拼自动生成修正版镜头卡。

## 7. Prompt 工作流标准

短剧工作流的提示词不要一次性写成长文小说，而要按以下结构输出：

```text
【短剧项目状态】
【一句话剧情与角色】
【角色资产 / 三视图】
【剧情目标】
【当前镜头提示词】
【下一段承接】
【需要补充的信息】
```

每段镜头必须包含：

```text
时长：
开场画面：
主体与场景：
大师级运镜：
表演调度：
时间轴：
0-2s：
2-5s：
5-8s：
8-10s：
台词 / 字幕：
声音：
视觉风格：
负面约束：
下一段承接：
```

## 8. 平台适配策略

### 8.1 即梦 / Seedance

默认中文输出，强调：

- 任务类型
- 参考素材
- 主体一致性
- 分镜
- 首尾帧
- 字幕和声音
- 负面约束

### 8.2 Veo

默认英文镜头语言，强调：

- Subject
- Action
- Scene
- Camera
- Temporal elements
- Audio
- Duration and aspect ratio

### 8.3 Runway

强调输入图 + 文本动作：

- 用角色图或首帧作为锚点。
- 提示词重点写动作和镜头，不重复描述图片已有内容。
- 避免否定式提示，尽量改成正向描述。

### 8.4 Sora

强调：

- 核心意图 + 镜头 + 动作 + 节奏。
- Storyboard 逐秒规划。
- Stitch / reorder / extend / reprompt / remix 连接多个片段。
- 多人对白要减少角色数量并明确时序。

## 9. 合规和隐私边界

啊拼短剧工作流必须坚持以下边界：

- 不自动调用视频平台 API。
- 不自动上传用户图片、角色图、剧情草稿。
- 图片外发必须尊重隐私设置。
- 不生成未成年人性化、低俗、违法、侵权、冒充真实人物或未经授权 IP 的内容。
- 不承诺发布、备案、引流或收益。
- 国内公开传播前，用户需要自行处理许可证、备案、平台审核、版权、肖像、AI 标识等问题。
