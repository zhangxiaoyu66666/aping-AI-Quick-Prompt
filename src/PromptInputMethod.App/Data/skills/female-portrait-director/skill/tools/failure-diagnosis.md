## 1. 工具定位

失败诊断工具用于分析用户已经生成失败、跑偏、不稳定、不符合预期的提示词、参数组合或图片结果，并给出明确原因、修复策略和可复制的修复版提示词。

本工具的核心不是泛泛说“提示词不够详细”，而是必须精准判断失败发生在哪个层级：

```text
参数锁定失败；
route 选择失败；
风格冲突；
服装描述不足；
五官模块缺失；
身形表达不安全或不稳定；
镜头构图冲突；
光线滤镜冲突；
场景信息过载；
动作链过复杂；
负面约束不足；
平台用途误导；
上传服装还原失败；
模型生成能力限制。
```

## 3. 工具职责

本工具负责：

```text
1. 读取用户的失败描述；
2. 读取用户提供的原提示词 / 参数 / 图片反馈；
3. 判断失败类型；
4. 找到失败原因；
5. 判断属于提示词问题、参数冲突、route 错误、负面约束不足，还是模型能力限制；
6. 给出优先级排序；
7. 提供具体修复策略；
8. 输出修复版提示词；
9. 输出增强版负面约束；
10. 给出后续质检清单。
```

本工具不负责：

```text
1. 只给泛泛建议；
2. 只说“多写细节”；
3. 不看用户原参数就随便重写；
4. 擅自改变用户原始风格；
5. 把失败归咎于用户；
6. 忽略模型能力边界；
7. 不输出修复版提示词；
8. 只诊断不修复。
```

## 5. 诊断总原则

失败诊断必须遵守以下原则：

```text
1. 先找主问题，再找次问题；
2. 先判断 route 是否正确；
3. 先判断用户明确参数是否被保护；
4. 先判断提示词是否存在冲突；
5. 先判断服装、五官、身形、镜头、光线是否足够具体；
6. 先判断负面约束是否覆盖失败点；
7. 能修复的必须给修复版；
8. 模型能力限制必须明确说明；
9. 不把所有问题都归结为“模型不行”；
10. 不输出空泛建议。
```

每次执行失败诊断时，按以下流程处理：

```text
第一步：提取用户反馈

第三步：判断任务类型
第四步：判断主 route 是否正确

第六步：检查风格冲突

第八步：检查服装 / 配色方向 / 华丽度 / 五官 / 身形 / 镜头 / 光线具体度
第九步：检查负面约束是否不足
第十步：判断模型能力限制
第十一步：输出失败原因
第十二步：输出修复策略
第十三步：输出修复版提示词
第十四步：输出质检清单
```

# 9. 具体失败诊断规则

---
## 9.1 F01：主 route 选错

表现：

```text
用户要新中式，结果像古风仙侠；
用户要电商主图，结果像普通写真；
用户要清纯生活照，结果变成纯欲；
用户要都市时尚，结果变成影楼棚拍；
用户要清冷古风，结果变成明媚华贵古风。
```

原因：

```text
风格关键词没有设置主次；
style-registry.md 没有正确分流；
平台用途被误认为主风格；
overlay 抢了主 route；
用户补充要求中的某个词权重过高。
```

诊断话术：

```text
主要问题是主 route 判断不稳。提示词里同时出现多个风格信号，模型没有明确知道应该优先执行哪一个，因此生成结果被其他风格词带偏。
```

修复策略：

```text
明确主 route；
将其他风格降级为气质、配色或局部细节；
在提示词前半段明确“整体风格为 xxx，不是 xxx”；
在负面约束中排除错误 route。
```

示例修复：

```text
整体风格必须锁定为新中式东方写真，重点是茶室、屏风、竹影、盘扣、东方留白，不是古风仙侠，不使用披帛、大袖衫、花钿、宫殿回廊等仙侠元素。
```

## 9.2 F02：用户参数未锁定

表现：

```text
用户写白红金，结果变成全红金；
用户写上半身，结果变成全身；
用户写短款吊带，结果变成连衣裙；
用户写冷白，结果变成暖黄；
用户写丹凤眼，结果变成圆眼甜妹。
```

原因：

```text
参数没有放在足够靠前的位置；
没有明确“必须保留”；
系统自动补全覆盖了用户参数；
提示词中存在更强的默认风格词；
负面约束没有排除替换项。
```

诊断话术：

```text
问题不是参数本身，而是参数没有被锁定。模型把你写的参数当作参考，而不是必须执行的约束，所以被后面的风格词或默认补全覆盖了。
```

修复策略：

```text
把用户参数放在提示词前半段；
使用“必须保留 / 不得替换 / 只能扩写不能改写”；
将错误替换项写入负面约束；
减少与用户参数冲突的默认描述。
```

示例修复：

```text
服装必须锁定为银灰色贴身吊带背心 + 浅灰色低腰短裤，不得替换成连衣裙、针织开衫、古风服装、宽松上衣或其他款式。
```

## 9.3 F03：风格关键词冲突

表现：

```text
清冷但又大面积红金喜庆；
新中式但又披帛大袖衫；
生活照但又影楼精修大片；
电商主图但又强胶片复古；
运动健康但又纯欲低机位；
法式慵懒但又赛博霓虹。
```

原因：

```text
多个强风格词同时出现；
没有主次关系；
冲突元素没有降级；
某些风格词语义方向相反；
用户补充要求没有被转译。
```

诊断话术：

```text
提示词中存在风格冲突。每个词单独看都成立，但组合后会给模型多个互相竞争的方向，导致画面不稳定或生成出折中但不准确的结果。
```

修复策略：

```text
确定主风格；
冲突词降级；
保留一个主色调；
只保留一个主场景；
明确排除错误风格。
```

示例修复：

```text
清冷古风中可以保留红色，但红色只能作为衣缘或花钿的小面积辅色，金色只能作为头饰或刺绣高光，主体仍以月白、冷白和低饱和冷调为主。
```

## 9.4 F04：提示词机械填参

表现：

```text
输出像字段列表；
没有画面事件；
人物像站桩；
场景像背景贴图；
光线没有落点；
画面没有故事感；
提示词和用户参数之间没有自然融合。
```

原因：

```text
只是把用户字段拼接在一起；
缺少导演式扩写；
缺少时间切片；
缺少姿态动作链；
缺少人物与环境关系。
```

诊断话术：

```text
这条提示词最大问题是机械填参。参数虽然都写了，但没有转化为一个可被拍摄的真实画面，所以模型只能按关键词随机拼接。
```

修复策略：

```text
加入时间切片；
加入主事件；
加入稳定动作链；
加入场景层次；
加入光线落点；
把字段改写成连续画面。
```

示例修复：

```text
不要写“咖啡馆、白色针织、自然光”，应写成“午后咖啡馆靠窗座位，人物自然坐在浅色木桌旁，柔和自然光从玻璃窗边进入，照亮脸侧、针织开衫纹理和桌面咖啡杯”。
```

## 9.5 F05：五官模块缺失或过泛

表现：

```text
每张脸都差不多；
人物像网红脸；
古风、港风、清纯、都市都长同一张脸；
眼型不稳定；
五官没有记忆点；
脸部风格和画面风格不一致。
```

原因：

```text
只写“美女 / 精致五官 / 高颜值”；
没有五官方向；
没有眼型、脸型、眉眼、唇形的风格描述；
没有排除网红脸和欧美脸；
route 默认五官没有被调用。
```

诊断话术：

```text
五官趋同的主要原因是五官模块过泛。只写“精致美女”会让模型回到默认网红脸，而不是根据风格生成差异化人物。
```

修复策略：

```text
加入明确五官方向；
根据 route 匹配五官；
补充脸型、眼型、眼神、鼻型、唇形；
在负面约束中排除网红脸、欧美脸、幼态脸。
```

五官方向参考：

```text
清纯生活照：初恋感淡颜 / 邻家清秀脸
纯欲曲线：清冷淡颜 / 邻家清秀脸
都市时尚：高级淡颜 / 轻熟都市脸
复古港风：明艳东方浓颜 / 港风故事脸
新中式：古典东方美人脸 / 清冷淡颜
古风仙侠：精致东方淡颜 / 东方淡颜丹凤眼
影楼精修：精致东方淡颜 / 明艳东方脸
东方丰腴：古典东方美人脸 / 温柔丰润脸
```

## 9.6 F06：身形表达不清或低俗化

表现：

```text
用户要曲线，结果没有身形吸引力；
用户要丰腴，结果变成夸张身材；
用户要纯欲，结果变成低俗私房；
线条重点变成身体局部特写；
胸部、腰、腿表达失控；
姿势低俗。
```

原因：

```text
只写“性感 / 曲线 / 身材好”；
没有成年边界；
没有服装完整；
没有规定通过姿态、服装和光线表达；
没有禁止局部特写；
镜头角度低俗。
```

诊断话术：

```text
身形表达失败通常不是因为写得不够“强”，而是没有规定表达方式。模型会把“性感、曲线”理解成局部特写或低俗姿态，所以需要转译为安全、稳定的整体线条表达。
```

修复策略：

```text
明确成年女性；
使用自然吸引力曲线；
通过服装版型、轻侧身、光线和构图呈现；
禁止低俗低机位和局部特写；
身形强度分级。
```

示例修复：

```text
身形具有自然成年女性吸引力，通过贴身但完整的日常穿搭、轻侧身姿态、大腿以上构图和柔和冷白光线呈现锁骨、腰线、小腹和大腿整体线条，不使用身体局部特写。
```

## 9.7 F07：服装描述不足

表现：

```text
服装变款；
颜色不对；
衣服结构混乱；
短款变长款；
吊带变背心；
针织变普通布料；
古风披帛消失；
电商图图案丢失。
```

原因：

```text
服装只写名称；
没有颜色、材质、版型、结构、装饰；
没有“必须保留”；
没有排除替换服装；
场景或风格词抢走服装优先级。
```

诊断话术：

```text
服装失败的根本原因是服装没有被结构化锁定。只写服装名称，模型会自动按风格补一件“类似衣服”，而不是严格保留你指定的款式。
```

修复策略：

```text
补充服装品类；
补充颜色；
补充材质；
补充版型；
补充领口、袖口、腰线、下摆；
补充装饰；
写明不得替换。
```

示例修复：

```text
服装必须锁定为雾蓝色短款吊带 + 白色轻薄开衫 + 浅色短裤。雾蓝色短款吊带应保持贴身短款版型，白色开衫自然垂落但不遮挡主体服装，浅色短裤腰线清楚，不得替换为连衣裙、长裤、宽松上衣或其他服装。
```

## 9.8 F08：上传服装还原失败

表现：

```text

F03：风格关键词冲突
F12：光线滤镜冲突
F14：负面约束不足
```

## 24. 最终原则

失败诊断工具的最终原则是：

```text
不是简单说“提示词不够详细”，而是要指出具体失败层级；
不是只诊断不修复，而是必须给出可执行修复版；
不是把所有失败都归咎于模型，而是区分提示词问题、参数冲突和模型能力限制；
不是重新创造新风格，而是在用户原目标上修复；
不是模糊建议，而是给出明确优先级、修复策略和可复制提示词。
```
