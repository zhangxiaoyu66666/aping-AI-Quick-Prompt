## 1. 工具定位

本工具用于将上传图片作为生成约束，在保留人物身份特征或产品核心视觉特征的基础上，直接生成用户选择的目标风格图片。

本工具默认不输出提示词。内部完成参考图锁定、风格路由、导演设计和安全检查后，调用可用的图片编辑、参考图生成或多图合成能力。

## 2. 与相邻能力的边界

| 用户目标 | 使用能力 |
| --- | --- |
| 分析图片、反推提示词 | `image-to-prompt` |
| 保留人物或产品，直接生成图片 | `reference-image-generate` |
| 上传服装并生成模特试穿图 | `reference-image-generate` 编排，`ecommerce-tryon` 主导服装还原 |

不得把本工具退化成普通文本生图，也不得在用户没有要求时先输出一大段提示词。

## 3. 触发条件

触发示例：

```text
保留我的五官，生成港风写真
用我的自拍做一张古风仙侠图
保持这个产品不变，生成一张高级广告图
保留我的脸，穿上第二张图里的衣服
把这个配饰戴到我的照片里
参考第一张图的人物，使用第二张图的产品，生成小红书种草图
不要提示词，直接出图
```

仅要求“分析图片”“反推提示词”“图片转提示词”时，不调用本工具，改用 `image-to-prompt`。

## 5. 执行流程

```text
第一步：识别每张上传图片的角色；
第二步：确认人物图片属于本人或已授权成年人物；
第三步：建立人物 / 产品锁定表；
第四步：识别用户明确要求改变和禁止改变的内容；
第五步：选择唯一目标 Route 和可选 Overlay；
第六步：通过 director-gate 完成内部导演设计；
第七步：生成内部提示词和负面约束；
第八步：调用图片编辑、参考图生成或多图合成能力；
第九步：检查五官身份和产品核心结构是否漂移；
第十步：必要时进行一次定向重试；
第十一步：返回图片和简短结果说明。
```

## 6. 图片角色识别

必须显式标注：

```text
identity_reference：本人或已授权成年人物图片
product_reference：服装、配饰、包装或普通实物
style_reference：只参考风格、构图、光线或氛围
edit_target：需要直接修改的原图
```

用户上传多张图但未说明用途时，先询问每张图片的角色，不自行猜测。

## 7. 默认锁定策略

### 7.1 人物

默认锁定：

```text
脸型；
眉眼；
鼻唇；
可识别五官特征；
人物身份可识别度。
```

默认允许风格化：

```text
发型；
妆容；
服装；
姿态；
场景；
镜头；
光线；
滤镜。
```

### 7.2 产品

默认锁定：

```text
品类；
轮廓；
颜色；
材质；
图案；
关键结构和装饰。
```

Logo 与小字尽量保留，但不承诺逐字准确。

## 8. 路由规则

```text
自拍 + 港风写真 → retro-hongkong
自拍 + 古风仙侠 → gufeng-xianxia 或用户明确指定的古风增强 Route
自拍 + 都市街拍 → urban-fashion
自拍 + 服装图 + 试穿 → ecommerce-tryon
自拍 + 配饰图 + 古风佩戴 → 目标古风 Route，配饰作为 product_reference 锁定
产品包装图 + 棚拍广告 → 选择最匹配的商业视觉方向，产品锁定优先
产品图 + 生活方式种草 → 目标生活方式 Route，产品锁定优先于氛围扩写
```

如果现有 Route 无法完整覆盖产品广告方向，仍可使用最接近的视觉 Route 作为导演参考；不得虚构未实现 Route。

## 9. 内部生成提示词要求

内部提示词必须明确写出：

```text
输入图片角色；
必须保留的人物身份特征；
必须保留的产品核心视觉特征；
允许改变的内容；
禁止改变的内容；
目标 Route 和 Overlay；
导演式场景、动作链、镜头、光线和滤镜；
安全边界；
保真检查重点。
```

## 10. 保真检查与重试

首次生成后检查：

```text
人物脸型、眉眼、鼻唇是否明显漂移；
人物是否仍可识别；
产品是否变成类似款；
产品颜色、轮廓、材质、图案或关键结构是否明显变化；
产品是否被手臂、头发、道具或构图遮挡；
目标风格是否覆盖了主体保真。
```

如果漂移明显，只允许进行一次定向重试：

```text
强化 identity_reference 的五官锁定；
强化 product_reference 的核心结构锁定；
降低不必要的风格化；
减少复杂动作和遮挡；
保留原目标 Route。
```

重试后仍无法满足时，明确说明限制，不静默切换为普通文本生图。

## 11. 安全与授权

允许：

```text
用户本人自拍；
已授权成年人物；
正常写真、换装、妆造和场景转换；
产品场景化；
服装试穿；
配饰佩戴；
广告图和种草图。
```

拒绝：

```text
未成年人或疑似未成年人物的性化转换；
未获授权的真实人物身份复刻；
明星、公众人物或第三方人物的欺骗性冒充；
偷拍、隐私侵犯、骚扰、诽谤；
色情深度伪造；
产品仿冒、虚假宣传或违法用途。
```

用户明确说“我的自拍”时视为本人授权。第三方人物图片授权不明确时，生成前询问确认。

## 12. 输出规则

默认输出：

```text
生成后的图片
简短说明：已保留的主体、应用的目标风格、是否进行了定向重试
```

默认不输出内部提示词。用户明确要求查看时，才输出内部生成提示词和负面约束。

## 13. 最终原则

```text
参考图不是普通灵感图，而是生成约束；
身份参考锁定人物五官；
产品参考锁定商品核心视觉；
Route 负责目标风格，不得覆盖锁定主体；
默认直接生成图片，不要求用户先整理提示词；
保真失败必须说明，不静默降级。
```
