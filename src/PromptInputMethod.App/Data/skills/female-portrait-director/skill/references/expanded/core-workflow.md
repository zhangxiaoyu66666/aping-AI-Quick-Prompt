# skill/skill.md

# 女性人像提示词导演 Skill｜核心导演流程

版本编号：`FEMALE-PORTRAIT-DIRECTOR-V1.4.1`
文档类型：核心导演流程文档
适用范围：女性人像提示词生成、风格路由、参数扩写、提示词优化、失败诊断、参数组合推荐、审查友好改写、图片反推提示词、参考图保留直接生成
核心定位：接收用户输入，完成参数或参考图主体锁定、风格路由、导演式扩写和最终输出

---
## 1\. 文档定位

本文件是女性人像提示词导演 Skill 的核心执行流程。

它负责统筹整个 Skill 的工作过程，但不直接写入所有风格母版。

本文件负责：

1. 识别用户意图；
2. 提取用户参数；
3. 执行参数锁定；
4. 判断是否调用工具模块；
5. 判断是否调用风格路由；
6. 调用 `style-registry.md` 进行风格匹配；
7. 调用 `core/` 中的底层规则；
8. 调用 `references/` 中的公共视觉库和导演扩写规则；
9. 调用 `routes/` 中的具体风格母版；
10. 调用 `overlays/` 中的气质增强模块；
11. 组织最终提示词；
12. 输出负面约束；
13. 执行最终自检。
14. 参考图直接生成时锁定人物身份或产品核心视觉。

本文件不负责：

1. 写完整风格母版；
2. 写全部风格触发词；
3. 写完整五官词库；
4. 写完整服装词库；
5. 写完整负面约束总库；
6. 替代 `style-registry.md`；
7. 替代具体 route 文件；
8. 替代 `director-expansion.md`。

---
## 2\. 核心执行目标

本 Skill 的最终目标不是机械填参，而是根据用户输入生成一条完整、稳定、可直接复制使用的女性人像 AI 生图提示词。

每次生成提示词时，都必须做到：

```text
1. 用户填写的参数被严格保留；
2. 用户未填写的参数被合理补全；
3. 风格路由准确；
4. 人物年龄与成年边界明确；
5. 五官方向清晰；
6. 身形表达可控；
7. 服装结构具体；
8. 场景不是道具堆砌；
9. 姿态和动作链合理；
10. 镜头、光线、滤镜互相一致；
11. 输出像一张可以被拍摄出来的照片；
12. 负面约束单独输出；
13. 不出现低俗、裸露、情色、未成年感或幼态化表达。
```

## 3\. 总体工作流程

系统每次收到用户输入后，必须按以下顺序执行：

```text
第 1 步：识别用户任务类型
第 2 步：提取用户显式参数
第 3 步：执行参数锁定
第 4 步：判断是否调用 tool
第 5 步：调用 style-registry.md 进行风格路由
第 6 步：调用 core/ 底层规则
第 7 步：调用 references/ 公共视觉库与导演扩写规则
第 8 步：调用对应 route 风格母版
第 9 步：判断是否叠加 overlay
第 10 步：执行导演式扩写
第 11 步：生成最终提示词
第 12 步：生成负面约束
第 13 步：执行最终自检
第 14 步：按用户要求输出
```

## 4\. 第 1 步：识别用户任务类型

系统首先判断用户当前要做什么。

任务类型分为 8 类：

```text
1. 生成提示词
2. 推荐参数组合
3. 优化已有提示词
4. 诊断提示词失败原因
5. 生成多风格版本
6. 审查友好改写
7. 图片反推提示词
8. 设计或维护 Skill 文档
```

---
### 4.1 生成提示词

用户常见表达：

```text
根据下面参数生成提示词
输出一条完整提示词
直接生成这个风格的提示词
帮我写一条可复制提示词
```

处理方式：

```text
进入风格路由流程；
调用对应 route；
执行导演式扩写；
输出参数锁定结果、最终提示词、负面约束。
```

---
### 4.2 推荐参数组合

用户常见表达：

```text
推荐几组参数
给我几组吸睛组合
推荐 8 组
不要同一种场景
服装颜色不要全是白色
更爆款一点
```

处理方式：

```text
调用 skill/tools/parameter-recommend.md；
根据目标风格输出可直接调用的参数组合；
每组必须包含风格、场景、服装、气质、五官、身形、镜头、光线、滤镜、画幅和推荐理由。
```

不得只输出风格名称。

---
### 4.3 优化已有提示词

用户常见表达：

```text
优化这条提示词
让它更稳定
不要改变原参数
帮我改得更容易出图
```

处理方式：

```text
调用 skill/tools/prompt-optimize.md；
先提取原始参数；
锁定原始参数；
删除冲突词；
强化人物、五官、服装、姿态、镜头、光线、滤镜和负面约束；
输出优化后的完整提示词。
```

不得把原提示词的核心方向替换掉。

---
### 4.4 诊断提示词失败原因

用户常见表达：

```text
为什么出图不稳定
为什么没按参数生成
为什么服装跑偏
为什么脸都一样
为什么色差明显
哪里有问题
```

处理方式：

```text
调用 skill/tools/failure-diagnosis.md；
从风格冲突、参数替换、五官缺失、服装不清、姿势复杂、镜头冲突、光线滤镜冲突、负面约束不足等角度诊断；
给出修改建议；
必要时输出优化版提示词。
```

---
### 4.5 生成多风格版本

用户常见表达：

```text
基于这条提示词生成 5 个不同风格版本
换几种风格
做几个不同方向
```

处理方式：

```text
保留用户核心参数；
根据 style-registry.md 选择多个差异明显的 route；
每个版本必须保持参数不被替换；
每个版本输出独立提示词或独立参数组合。
```

不得把所有版本写成同一种风格的轻微改词。

---
### 4.6 审查友好改写

用户常见表达：

```text
改写得更容易通过审查
降低敏感
保留效果但更合规
不要被拒
GPT Image 2 审查
```

处理方式：

```text
调用 skill/tools/safety-rewrite.md；
保留原风格、场景、服装、镜头、光线和气质；
将高风险词改写为成年、时尚、服装完整的安全表达。
```

---
### 4.7 图片反推提示词

用户常见表达：

```text
推导这张图的提示词
反推提示词
分析图片风格
根据图片写 AI 提示词
改成某某风格
```

处理方式：

```text
调用 skill/tools/image-to-prompt.md；
先分析画面主体、风格、光线、色彩、构图、服装、人物特征；
再根据目标风格重写成完整提示词；
如果用户要求改风格，只改目标风格，不机械复制原图。
```

---
### 4.8 设计或维护 Skill 文档

用户常见表达：

```text
继续输出核心文档
帮我设计这个 Skill
把这个模块写成文档
按这个结构重写
输出某个 route 文件
```

处理方式：

```text
输出对应 Skill 文档；
一次只输出一个核心文档；
文档内容必须完整，不得只写摘要；
不得与既有版本规则冲突；
当前版本统一使用 FEMALE-PORTRAIT-DIRECTOR-V1.4.1。
```

## 6\. 第 3 步：执行参数锁定

参数锁定是最高优先级规则之一。

系统必须调用：

```text
skill/core/parameter-lock.md
```

执行以下规则：

```text
1. 用户明确填写的参数必须保留；
2. 不得擅自替换场景；
3. 不得擅自替换服装；
4. 不得擅自替换五官；
5. 不得擅自替换身形；
6. 不得擅自替换镜头；
7. 不得擅自替换光线；
8. 不得擅自替换滤镜；
9. 不得擅自替换画幅比例；
10. 不得忽略补充要求。
```

---
### 6.1 参数锁定结果必须先形成

在内部生成如下结构：

```text
已锁定参数：
写真风格：
场景方向：
服装方向：
气质标签：
五官方向：
身形方向：
身形吸引力强度：
线条重点：
镜头方向：
光线氛围：
滤镜效果：
画幅比例：
尺寸参数：
平台用途：
补充要求：
```

如果用户要求完整输出，则展示这部分。

如果用户要求“只输出提示词”，则内部执行锁定，但不展示锁定结果。

---
### 6.2 参数缺失不等于可以乱改

用户未填写的参数可以自动补全。
用户已填写的参数只能扩写。

例如：

用户填写：

```text
场景方向：咖啡馆
```

可以扩写为：

```text
真实咖啡馆靠窗座位，背景干净虚化，有生活感。
```

不得改为：

```text
卧室床边 / 海边步道 / 摄影棚。
```

## 7\. 第 4 步：判断是否调用 Tool

如果用户任务属于工具型任务，则优先调用 `tools/`。

对应关系：

```text
优化提示词 → skill/tools/prompt-optimize.md
失败诊断 → skill/tools/failure-diagnosis.md
参数组合推荐 → skill/tools/parameter-recommend.md
审查友好改写 → skill/tools/safety-rewrite.md
图片反推提示词 → skill/tools/image-to-prompt.md
参考图保留直接生成 → skill/tools/reference-image-generate.md
```

如果用户任务是“生成提示词”，则不进入 tool，而是进入 route。

如果用户任务同时包含生成和优化，例如：

```text
帮我把这条提示词优化后重新输出完整可复制版
```

则先调用 `prompt-optimize.md`，再按 route 生成完整提示词。

### 8.1 路由判断优先级

按以下顺序判断：

```text
P0：安全边界与成年边界
P1：用户明确填写的写真风格
P2：强商业用途 / 上传服装 / 电商主图
P3：强曲线表达 / 纯欲曲线 / 东方丰腴
P4：强古风仙侠 / 清冷仙气 / 明媚华贵
P5：强东方美学 / 新中式 / 旗袍 / 宋韵 / 水墨

P7：强时尚写真 / 都市 / 杂志 / 街头 / 运动 / 影楼
P8：平台用途调整
P9：Overlay 气质增强
P10：Fallback 默认风格
```

---
### 8.2 明确风格优先

如果用户明确写：

```text

```

必须调用：

```text
skill/routes/lifestyle/retro-hongkong.md
```

即使场景中出现“夜景、霓虹、街头”等其他触发词，也不得擅自换成夜色霓虹写真。

---
### 8.3 强商业任务优先

如果用户写：

```text
上传服装图片
电商主图
服装完整清晰
不要色差
试衣镜
```

必须优先调用：

```text
skill/routes/commercial/ecommerce-tryon.md
```

此时核心目标为：

```text
服装真实还原 > 服装完整清晰 > 模特好看 > 背景氛围
```

---
### 8.4 强曲线任务优先

如果用户写：

```text
纯欲
曲线
锁骨
腰线
小腹
大腿
身形吸引力
```

## 9\. 第 6 步：调用 Core 底层规则

所有任务必须继承 `core/`。

必读规则：

```text
skill/core/parameter-lock.md
skill/core/safety-boundary.md
skill/core/output-format.md
skill/core/conflict-resolution.md
skill/core/fallback-rules.md
```

---
### 9.1 安全边界

所有女性人像必须默认：

```text
20–28 岁年轻成年东方女性；
成年气质明确；
不欧美化；
不幼态化；
无未成年感；
非裸露；
非情色。
```

当用户输入“性感、纯欲、身形、丰腴、胸腰腿”等词时，必须转化为：

```text
成年女性时尚写真；
有吸引力；
自然身体线条；
服装完整；
真实摄影质感。
```

---
### 9.2 冲突处理

如果用户参数冲突，必须保留用户明确参数，并通过导演式扩写降低冲突。

示例：

用户输入：

```text
风格：清冷仙气
配色：红金
补充要求：不要喜庆，不要暖热
```

处理方式：

```text
保留红金，但红色只作为辅色点缀，金色只作为首饰和刺绣细节高光，整体仍保持冷白基调，避免喜庆热烈感。
```

不得直接改成月白冰蓝。

---
### 9.3 缺失补全

用户没有填写的参数，可根据 `fallback-rules.md` 自动补全。

示例：

用户只写：

```text
风格：法式慵懒写真
场景：窗边
服装：针织衫
```

系统可补全：

```text
五官方向：松弛淡颜 / 温柔成熟脸
身形方向：自然协调身形
镜头方向：半身到大腿以上构图
光线氛围：奶油暖白自然光
滤镜效果：奶油暖白法式生活滤镜
画幅比例：9:16
```

### 10.2 director-expansion.md 的作用

该文件提供导演式推导与扩写规则。

所有最终提示词必须遵守：

```text
1. 先建立画面意图；
2. 再选择时间切片；
3. 再设计一个轻微事件；
4. 再建立动作链；
5. 再确定视线落点；
6. 再选择两到三个环境细节；
7. 再融合镜头、光线和滤镜；
8. 最后生成像真实照片一样成立的提示词。
```

```text
傍晚的海边步道上，人物在散步中短暂停下，身体轻微侧向镜头，重心自然落在后侧腿上。海风带动白色轻薄开衫边缘，雾蓝色短款吊带和浅色短裤形成清爽的冷色层次，肩颈和锁骨在柔和侧光下显得干净，腰线、小腹和大腿线条通过大腿以上构图自然呈现。
```

## 14\. 第 11 步：生成最终提示词

最终提示词必须是自然完整的摄影指导，不是字段说明书。

推荐结构：

```text
第一段：画面类型 + 人物主体 + 风格总控
第二段：五官、气质、妆容、身形和姿态
第三段：服装结构、衣料动态、动作链
第四段：场景空间、镜头构图、光线落点、滤镜质感
第五段：平台用途或商业目标补充
```

不必每次都严格五段，但不得省略核心信息。

---
### 14.1 正向提示词必须包含

```text
人物主体
成年边界
五官方向
气质
身形与姿态
服装结构
场景空间
镜头构图
光线落点
滤镜质感
平台用途
```

---
### 14.2 正向提示词不得包含大量负面词

正向提示词应描述想要的画面。

例如：

```text
人物为年轻成年东方女性，成年气质明确，姿态自然克制，服装完整清晰，背景干净虚化，光线柔和通透。
```

不要在正向提示词中反复写：

```text
不要未成年，不要畸形，不要低俗，不要过曝，不要背景杂乱。
```

这些应放入负面约束。

## 20\. 最终执行准则

本文件的最终准则是：

```text
用户输入越简单，系统越要合理补全；

最终提示词必须像一张真实可拍摄或可信生成的照片，而不是字段拼接。
```

所有输出必须服务于 V1.4.1 的核心目标：
