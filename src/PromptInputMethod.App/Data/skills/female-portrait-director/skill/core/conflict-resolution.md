## 1. 文档定位

本文件用于处理女性人像提示词导演 Skill 中的参数冲突问题。

参数冲突是导致出图不稳定、风格串味、服装跑偏、画面低俗、人物不一致、平台目标失效的主要原因之一。

本文件负责规定：

1. 什么是参数冲突；
2. 冲突出现时的处理优先级；
3. 哪些参数不能被替换；
4. 哪些参数可以被弱化或转译；
5. 风格冲突如何处理；
6. 场景冲突如何处理；
7. 服装冲突如何处理；
8. 身形与安全冲突如何处理；
9. 光线与滤镜冲突如何处理；
10. 平台用途与主风格冲突如何处理；
11. route、overlay、tool 同时作用时如何避免互相覆盖。

## 2. 总原则

参数冲突处理的最高原则是：

```text
先保留用户明确参数；
再判断画面主目标；
再降低冲突强度；
再用导演式扩写融合；
最后用负面约束防止跑偏。
```

禁止：

```text
直接把用户参数改成系统默认参数；
直接删除用户填写的关键需求；
为了稳定出图而换场景、换服装、换风格；
为了安全而完全删掉用户原本的画面目标；
让平台用途覆盖主风格；
让 overlay 替代 route；
让 route 默认值覆盖用户输入。
```

允许：

```text
保留用户参数；
调整表达方式；
降低冲突词强度；
把高风险词转译为安全表达；
缩小颜色占比；
减少道具数量；
降低背景复杂度；
调整镜头目的；
用光线和构图统一画面。
```

## 3. 冲突判断优先级

当多个参数冲突时，按以下优先级处理：

```text
P0：安全边界与成年边界
P0.5：参考图人物身份或产品核心视觉锁定
P1：用户明确填写的核心参数
P2：用户补充要求
P3：任务主目标
P4：写真风格 / route
P5：服装方向
P6：场景方向
P7：五官与气质
P8：身形与线条重点
P9：镜头方向
P10：光线氛围
P11：滤镜效果
P12：平台用途
P13：route 默认值
P14：overlay 气质增强
```

说明：

```text
安全边界永远最高；
用户明确参数高于 route 默认值；
补充要求通常是用户修正失败问题的关键，优先级很高；
平台用途只做构图和传播目标调整，不覆盖主风格；
overlay 只增强气质，不替代主 route。
参考图主体锁定高于 Route、Overlay 和默认补全。
```

# 5. 风格冲突处理

## 5.1 多个主风格同时出现

用户可能输入：

```text
清纯生活照 + 复古港风 + 新中式 + 纯欲曲线
```

处理规则：

```text
1. 如果用户明确指定“写真风格”字段，优先锁定该字段；
2. 其他风格词作为气质、滤镜或局部氛围处理；
3. 不允许多个主风格平均混合；
4. 必须确定一个主 route；
5. 其余风格只能作为辅助修饰。
```

错误处理：

```text
生成清纯生活照、复古港风、新中式、纯欲曲线混合风。
```

正确处理：

```text
主风格锁定为用户明确填写的风格；
其他词只作为色彩、气质或局部氛围补充。
```

---

## 5.2 清纯生活照 + 纯欲曲线

输入示例：

```text
写真风格：清纯生活照
气质：纯欲、有吸引力
线条重点：锁骨、腰线、小腹、大腿
```

冲突点：

```text
清纯生活照偏自然生活感；
纯欲曲线强调身形线条；
两者可以兼容，但必须避免低俗化。
```

处理方式：

```text
如果用户明确写“写真风格：清纯生活照”，保持清纯生活照为主 route；
将纯欲和线条重点转译为成年女性自然身体线条和克制吸引力；
镜头可采用半身到大腿以上构图，但不能低俗。
```

如果用户没有明确主风格，只强烈强调曲线：

```text
优先调用 routes/curve/pure-desire-curve.md。
```

```text
新中式偏现实东方人像；
```

```text
古风仙侠，唐风披帛，宫殿回廊，东方幻想角色海报。
```

```text
红金盛大、宫廷、明艳、华贵、女主感
```

调用：

```text
bright-luxury-gufeng。
```

# 6. 场景冲突处理

## 6.1 电商服装图 + 复杂街头

输入示例：

```text
风格：电商服装模特图
场景：街头
补充要求：服装完整清晰，不要复杂背景，不要色差
```

冲突点：

```text
电商图要求服装清晰；
街头场景容易背景复杂、色彩干扰、服装不清。
```

处理方式：

```text
保留街头场景，但将其处理为干净的轻商业街拍背景；
背景明显虚化，减少行人、招牌、车辆；
光线均匀，服装颜色准确；
人物姿态稳定，服装完整清晰。
```

禁止：

```text
直接改成摄影棚白底；
或者保留复杂街头导致服装被淹没。
```

---

```text
电影生活剧照感，卧室，影楼精修，真实生活感。
```

冲突点：

```text
生活剧照强调真实自然；
影楼精修强调布光、修饰和商业质感。
```

处理方式：

```text
如果用户主风格是生活剧照感，则保留真实卧室和自然姿态；
将影楼精修转译为画质清晰、肤质干净、光线稳定；
不做强棚拍背景和商业摆拍。
```

如果用户明确写：

```text
写真风格：影楼精修写真
```

则调用：

```text
studio-retouched.md。
```

```text
古风仙侠美人图，咖啡馆场景。
```

冲突点：

```text
古风仙侠服装与现代咖啡馆场景冲突。
```

处理方式：

```text
如果用户明确要求该混搭，保留现代咖啡馆，但将其处理为极简、弱化、背景虚化的空间；
重点仍放在古风人物、服装、妆造上；
```

更稳定的处理：

```text
可将咖啡馆转译为“带木质窗格与茶室感的安静室内空间”，但不得完全替换用户场景。
```

禁止：

# 7. 服装冲突处理

## 7.1 用户服装 vs route 默认服装

规则：

```text
用户服装永远优先于 route 默认服装。
```

示例：

route 默认法式慵懒常用：

```text
白衬衫、针织衫、浅色半裙
```

用户输入：

```text
服装：黑色针织背心 + 牛仔裤
```

正确处理：

```text
保留黑色针织背心和牛仔裤；
通过窗边自然光、松弛姿态、法式公寓氛围来保持法式慵懒感。
```

禁止：

```text
改成白衬衫和米白半裙。
```

---

## 7.2 服装展示 + 姿态遮挡

输入示例：

```text
电商服装模特图，要求服装完整清晰，姿势自然，可以手拿包。
```

冲突点：

```text
包袋可能遮挡服装主体。
```

处理方式：

```text
如果需要包袋，只能放在身体侧边或手自然下垂的位置；
不得遮挡领口、胸前、腰线、图案、下摆和版型重点。
```

---

## 7.3 古风服装 + 现代礼服化

输入示例：

```text
华丽唐风大袖衫，精修女主感，服装要高级。
```

冲突风险：

```text
“高级”容易被模型理解为现代礼服。
```

处理方式：

```text
明确服装属于唐风幻想古装体系；
高级体现为刺绣、披帛、腰封、珠玉头饰、面料层次；
不得出现西式礼服、现代晚礼服结构。
```

---

## 7.4 纯欲服装 + 不低俗

输入示例：

```text
服装：贴身吊带 + 短裤
补充要求：不要低俗
```

冲突点：

```text
贴身、短款服装容易被模型误读为低俗。
```

处理方式：

```text
将服装定义为生活化时尚穿搭；
强调服装完整、自然、不透明、不情趣化；
通过肩颈、锁骨、腰线和腿部比例表现线条，而不是通过暴露表现。
```

---

## 7.5 旗袍 + 曲线明显

输入示例：

```text
旗袍东方写真，曲线明显一点。
```

冲突点：

```text
旗袍曲线表达容易滑向低俗或情趣化。
```

处理方式：

```text
保留旗袍的东方含蓄曲线；
强调版型合体但端庄；
开衩克制；
姿态端正；
光线柔和；
整体复古、精致、东方美学。
```

禁止：

```text
过度开衩、低俗姿态、情趣旗袍、身体局部凝视。
```

# 8. 气质冲突处理

## 8.1 清冷 + 甜美

输入示例：

```text
气质：清冷、甜美、疏离、元气
```

冲突点：

```text
清冷疏离与甜美元气方向相反。
```

处理方式：

```text
判断主风格：
如果主风格是清冷仙气、新中式、清冷生活照，则保留清冷、疏离，甜美弱化为柔和；
如果主风格是活力运动、日系清透，则保留元气、甜美，清冷弱化为清爽。
```

---

## 8.2 法式慵懒 + 冷艳御姐

输入示例：

```text
法式慵懒写真，冷艳御姐气质。
```

处理方式：

```text
保留法式慵懒主风格；
叠加冷艳御姐 overlay；
人物姿态仍然松弛自然，但眼神更冷静，穿搭更简洁有气场，色调略降低甜度。
```

禁止：

```text
改成夜色霓虹冷艳都市写真。
```

---

## 8.3 明艳 + 清冷

输入示例：

```text
明艳但清冷。
```

处理方式：

```text
可以理解为“五官存在感较强，但表情和气场克制”；
不要同时输出热烈甜美和冷白疏离；
根据主风格选择明艳或清冷为主。
```

示例扩写：

```text
五官更有存在感，妆容精致，但眼神保持安静克制，整体不是热烈甜美，而是明艳中带冷感。
```

# 10. 身形与安全冲突处理

## 10.1 纯欲 + 强身形吸引力

输入示例：

```text
纯欲曲线，身形吸引力强，突出胸腰腿。
```

冲突点：

```text
强身形表达容易越界。
```

处理方式：

```text
保留身形吸引力强；
必须强化成年女性、服装完整、时尚写真感；
通过姿态、服装版型、光线落点和构图表现线条；
不得输出裸露、挑逗、情趣化、身体部位特写。
```

---

## 10.2 东方丰腴 + 古风

输入示例：

```text
古风仙侠，东方丰腴，胸腰腿线条明显。
```

处理方式：

```text
保留古风仙侠主风格；
将东方丰腴转译为成熟、柔润、协调的古典身形；
通过腰封、衣料垂坠、肩颈和整体轮廓表现曲线；
不得过度暴露，不得低俗化。
```

---

## 10.3 运动线条 + 低俗风险

输入示例：

```text
运动写真，突出腹部和腿。
```

处理方式：

```text
保留健康运动线条；
用全身或大腿以上构图表现运动后的健康状态；
服装为完整运动穿搭；
镜头避免身体部位特写；
整体阳光、清爽、健康。
```

# 14. 色彩冲突处理

## 14.1 清冷 + 红金

规则：

```text
清冷红金必须控制红色和金色占比。
```

处理方式：

```text
白色 / 月白 / 冷白为主色；
红色为小面积辅色；
金色为细节高光；
光线保持冷调；
避免喜庆、热烈、婚庆感。
```

---

## 14.2 新中式 + 多色堆叠

输入示例：

```text
新中式东方写真，红、绿、金、蓝、紫都要有。
```

冲突点：

```text
新中式强调克制、留白、统一色彩。
```

处理方式：

```text
保留用户颜色意图，但只选择 1 个主色、1 个辅色、1 个点缀色；
其余颜色转为极弱背景细节或不进入最终画面。
```

---

## 14.3 古风华丽 + 配色过多

规则：

```text
古风华丽不等于颜色越多越好。
```

处理方式：

```text
每次只允许 1 个主色、1 个辅色、1 个金属或珠宝点缀色；
华丽感通过材质、刺绣、头饰、层次和光线体现。
```

---

## 14.4 高华丽度 + 元素堆叠

规则：

```text
华丽度高不等于所有装饰同时加满。
用户填写的华丽度必须保留，但要转译为有层级的视觉密度。
```

处理方式：

```text
保留一个服装结构重点、一个发饰重点和一个材质高光重点；
背景可以华丽，但必须虚化并降低信息密度；
用刺绣密度、衣料质感、珠宝高光和层次关系表达华丽度；
不要通过无限增加首饰、道具、颜色和背景装饰表达华丽。
```

## 22. 最终原则

本文件的最终原则是：

```text
冲突不是让系统替用户改需求的理由；
冲突是让系统进行导演式融合和稳定化处理的理由。
```

判断冲突处理是否成功的标准是：

```text
用户明确参数仍然存在；
画面主目标更清晰；
冲突细节被合理降级；
风格没有串味；
安全边界没有丢失；
最终提示词仍像一张可以被拍摄或可信生成的照片。
```

所有模块必须遵守本文件。
