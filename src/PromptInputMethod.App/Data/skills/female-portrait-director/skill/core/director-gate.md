# 女性人像提示词导演 Skill｜导演模式强制门禁

版本编号：`FEMALE-PORTRAIT-DIRECTOR-V1.4.1`

本文件是所有提示词生成、优化、图片反推、参数推荐和直接生图任务的必经步骤。它用于保证系统先完成画面设计，再进行扩写，而不是机械拼接字段。

## 两阶段执行

### 阶段一：内部导演设计

在写最终提示词前，先在内部完成以下决策：

```text
1. 锁定用户显式参数；
2. 上传参考图时，按 reference-image-lock 建立图片角色表和主体锁定表；
3. 确定唯一主 Route 和可选 Overlay；
4. 判断画面主目标；
5. 选择一个明确时间切片；
6. 设计一个轻微主事件；
7. 建立连贯动作链：重心、肩颈、手部、衣料动态、头部方向、视线落点；
8. 锁定或补全配色方向：主色、辅色、点缀色、占比和冷暖关系；
9. 锁定或补全华丽度，并落实为服装层次、刺绣密度、发饰首饰等级和背景复杂度；
10. 选择两到三个环境细节；
11. 决定画幅比例、用户明确填写的尺寸参数、景别、机位、景深和构图目的；
12. 决定光线方向、落点、阴影和材质效果；
13. 决定滤镜色温、饱和度、对比度、高光、暗部和质感；
14. 执行成年边界和重点部位裸露检查；
15. 执行参数传递审计：逐项确认用户显式字段已经按生成作用进入最终提示词，尤其检查画幅比例、像素尺寸、分辨率或其他尺寸要求；
16. 检查扩写是否覆盖人物身份或产品核心结构；
17. 检查所有元素是否属于同一个可拍摄瞬间。
```

阶段一是内部工作，不输出冗长思维过程。除非用户明确要求简洁输出，标准模式必须输出经过整理的公开导演设计说明：使用可读段落说明年龄与五官、身形与仪态、服装配色与发饰、华丽度的可见设计、场景、镜头、光线与滤镜的最终摄影效果。不得只输出八行摘要或机械字段回显。导演详版可以进一步说明参数扩写前后的关系、选择性环境细节和构图目的，但同样不得暴露内部逐步推理文本。

### 阶段二：融合扩写

只有阶段一完成后，才允许生成最终提示词。最终提示词必须把人物、服装、动作、空间、镜头、光线和滤镜融合为同一个画面，不得退化为字段列表或关键词堆叠。

## 输出前门禁

以下任一项不成立时，重新设计后再输出：

```text
- 是否只有一个主 Route？
- 标准详细输出的参数锁定结果是否逐字段完整回显用户显式参数，没有合并、改写或省略？
- 人物设定、身形吸引力强度、线条重点等容易融入叙述的字段是否仍在参数锁定结果中单独列出？
- Route、Overlay 和补充方向是否仅作为附注，没有替代任何原始锁定字段？
- 用户没有明确要求简洁模式时，是否使用默认标准详细模式，没有自行压缩输出？
- 默认标准详细模式的最终提示词是否严格为五段，并分别覆盖人物、动作、服装、场景、光线五类职责？
- 五段是否均为实质扩写，没有短摘要、标签堆叠或段落职责合并？
- 向用户展示最终提示词和负面约束时，是否分别使用两个独立的 Markdown `text` 代码块，并把标题放在代码块外？
- 可复制代码块中是否只包含平台可用内容，没有混入解析、注释、Route、Overlay 或补充说明？
- 是否执行参数传递审计，确认每个用户显式字段不仅在锁定区出现，还按生成作用落实到最终提示词？
- 最终提示词第一句是否明确写出画幅比例，并在用户提供时保留像素尺寸、分辨率、宽高或其他尺寸要求？
- 用户未提供像素尺寸或分辨率时，是否避免虚构具体像素值？
- 用户填写的配色方向是否被完整保留并贯穿服装、场景、光线和滤镜？
- 用户填写的华丽度是否已转译为可见设计，而不是只重复标签？
- 上传参考图时，图片角色是否明确？
- 参考图扩写是否保留人物身份和产品核心结构？
- 是否存在一个明确时间切片？
- 是否只有一个轻微主事件？
- 动作链是否自然且可拍摄？
- 镜头和光线是否服务于画面主目标？
- 是否避免无关细节堆砌？
- 是否明确成年？
- 是否避免重点部位裸露和露骨性行为？
- 最终提示词是否是一张完整照片，而不是字段复述？
```

完整导演式扩写方法按需读取 [../references/director-expansion.md](../references/director-expansion.md)。

参考图主体保真规则按需读取 [reference-image-lock.md](reference-image-lock.md)。
