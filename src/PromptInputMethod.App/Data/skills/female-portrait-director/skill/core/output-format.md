# skill/core/output-format.md

# 女性人像提示词导演 Skill｜输出格式规则

版本编号：`FEMALE-PORTRAIT-DIRECTOR-V1.4.1`
文档类型：核心规则文档
适用范围：提示词生成、参数组合推荐、提示词优化、失败诊断、审查友好改写、图片反推提示词、Skill 文档输出
核心职责：统一所有输出格式，确保输出内容可复制、可执行、结构稳定、不混乱

---

## 1. 文档定位

本文件用于规定女性人像提示词导演 Skill 的所有输出格式。

它负责解决以下问题：

1. 不同任务输出格式不统一；
2. 提示词生成时只复述参数，没有形成完整画面；
3. 参数组合推荐不够完整，无法直接调用；
4. 提示词优化只给建议，不给最终可复制版本；
5. 失败诊断只分析问题，不给修复结果；
6. 审查友好改写过度删减用户画面目标；
7. 图片反推提示词只描述图片，不输出可用提示词；
8. Skill 文档输出结构不一致；
9. 正向提示词和负面约束混在一起；
10. 输出内容过于摘要化，无法直接用于生成图片。

本文件必须被以下模块继承：

```text
skill/skill.md
skill/style-registry.md
skill/core/parameter-lock.md
skill/core/safety-boundary.md
skill/core/conflict-resolution.md
skill/core/fallback-rules.md
skill/references/director-expansion.md
skill/routes/
skill/overlays/
skill/tools/
```

## 2. 总输出原则

所有输出必须遵守以下原则：

```text
1. 用户要求生成提示词时，必须直接输出可复制提示词；
2. 用户要求完整模式时，必须包含参数锁定结果、最终提示词、负面约束；
3. 用户要求只输出提示词时，不额外分析；
4. 用户要求推荐参数组合时，每组参数必须完整可调用；
5. 用户要求优化时，必须给出优化后的最终版本；
6. 用户要求诊断时，必须指出问题、原因、修改方式，并尽量给出修复版；
7. 用户要求审查友好改写时，必须保留原画面目标；
8. 用户要求图片反推提示词时，必须先分析再输出可复制提示词；
9. 用户要求 Skill 文档时，一次只输出一个核心文档；
10. 不得只写摘要，不得只写说明，不得让用户二次整理后才能使用。
11. 任何最终提示词都必须先通过 `core/director-gate.md` 的内部导演设计，不得边拼字段边输出。
12. 向用户展示最终提示词和负面约束时，必须分别使用两个独立的 Markdown `text` 代码块，保证可以直接复制；标题必须位于代码块外。
13. 输出前必须执行参数传递审计。`画幅比例`、像素尺寸、分辨率或其他尺寸要求必须进入最终提示词第一句；不得只停留在参数锁定结果中。
```

本 Skill 支持以下输出模式：

```text

4. 提示词优化模式

10. 简短说明模式
11. 直接生成图片模式
12. 参考图直接生成模式
```

系统必须先识别用户任务类型，再选择对应输出格式。


## 4.2.1 默认详细程度

完整提示词模式默认使用逐段导演式模块解析，不得压缩为字段回显。

参数锁定结果与导演式模块解析承担不同职责：

```text
参数锁定结果：
逐字段完整回显用户显式输入。不得合并、改写、省略，不得使用 Route、Overlay、补充方向或后文叙述替代原始字段。用户未填写的字段可以省略或标注“未指定”；系统补全内容必须单独标注为“补充方向”。

导演式模块解析：
将锁定参数融合扩写为可读的公开导演说明。不得退化为机械字段复述。
```

“不得机械填参”只约束导演式模块解析和最终提示词，不得被用于压缩参数锁定结果。Route 和 Overlay 如需展示，只能作为附注列出，不能替代写真风格、气质标签、人物设定或其他锁定字段。

默认标准详细模式中的最终提示词固定为五段式详细扩写，不得缩减为三段或四段，不得把多个段落职责合并为短摘要：

```text
第 1 段：首句写明画幅比例和用户明确填写的尺寸参数，再扩写人物、年龄、成年边界、五官、妆容、气质。
第 2 段：时间切片、轻微主事件、姿态、动作链、视线落点。
第 3 段：身形方向、线条重点、服装结构、配色、材质、发饰首饰、华丽度的可见设计。
第 4 段：场景、两到三个环境细节、前中后景关系、镜头、构图、景深。
第 5 段：光源方向、光线落点、高光、阴影、滤镜、色彩、质感。
```

每一段都必须有实质扩写，不能只有标签、短句或重复参数。只有用户明确要求简洁模式、简短版或压缩版时，才允许减少段落数量或合并段落职责。

## 4.2.2 参数传递审计硬规则

参数锁定结果只是第一层检查。完成最终提示词后，必须在内部执行参数传递审计，确认每个用户显式字段已经按照生成作用进入最终提示词。不得因为字段已在锁定区、解析区或 Route 母版中出现，就省略它在可复制最终提示词中的表达。

尺寸参数属于强制生成控制项：

```text
画幅比例：必须写入最终提示词第一句。
像素尺寸 / 分辨率 / 宽高：用户填写时，必须按原值写入最终提示词第一句；用户未填写时，不得虚构具体像素值。
方向：根据比例写明竖版、横版或方形。无法可靠判断时，写“{画幅比例} 画幅”。
```

最终提示词第一句固定采用以下结构：

```text
生成一张 {画幅比例} {竖版 / 横版 / 方形}、{像素尺寸或分辨率，如有}、{写真风格与质感} 的女性人像。
```

参数传递审计至少覆盖：

| 参数类别 | 参数锁定结果 | 导演式模块解析 | 最终提示词 |
| --- | --- | --- | --- |
| 画幅比例、像素尺寸、分辨率 | 必须出现 | 在镜头模块说明构图影响 | 必须在第一句明确出现 |
| 写真风格、人物设定、年龄成年边界 | 必须出现 | 在画面目标、年龄与五官模块扩写 | 必须明确出现 |
| 场景、服装、配色、华丽度 | 必须出现 | 在对应模块扩写 | 必须落实为可见画面 |
| 气质、五官、身形、吸引力强度、线条重点 | 必须出现 | 在人物与仪态模块扩写 | 必须落实为人物、姿态、衣料或光线表达 |
| 镜头、光线、滤镜 | 必须出现 | 在对应模块扩写 | 必须明确出现 |
| 平台用途、补充要求 | 用户填写时必须出现 | 说明其对构图或限制的影响 | 影响生成时必须落实 |

如果任意用户显式字段缺少落点，必须补齐后再输出。

## 4.2.3 可复制 Markdown 硬规则

向用户展示“最终提示词”和“负面约束”时，必须使用两个彼此独立的 Markdown fenced code block，并标注语言为 `text`。标题必须放在代码块外。代码块中只能包含可直接复制到生图平台的内容，不得混入解析、注释、Route、Overlay、补充方向或使用说明。

固定格式：

````markdown
三、最终提示词

```text
五段式最终提示词正文
```

四、负面约束

```text
负面约束正文
```
````

无论默认标准详细模式、用户明确要求的简洁模式、只输出提示词模式、优化修复版、安全改写版或图片反推提示词模式，只要向用户展示最终提示词和负面约束，就必须遵守此格式。直接生成图片模式可以只在内部准备提示词，不要求向用户展示代码块。

模块解析的职责是把参数扩写成用户可读的公开导演说明：

```text
年龄与五官：
说明可见年龄特征、成年边界、脸型、眉眼、眼神、鼻唇、妆容和气质效果。

身形与仪态：
说明自然比例、身体重心、肩颈状态、手部动作、衣料动态、头部方向和视线落点。

服装、配色与发饰：
说明颜色、材质、版型、层次、刺绣或纹理、配饰结构，以及服装如何服务动作和风格。必须解释配色方向中的主色、辅色和点缀色如何分配。

华丽度的可见设计：
说明华丽度如何落实为刺绣密度、材质精致度、服装层次数量、发饰首饰等级、珠宝高光数量和背景装饰复杂度，并指出需要克制的元素。

场景：
说明人物在空间中的位置，并选择两到三个环境细节建立前中后景关系。

镜头：
说明画幅、景别、机位、主体占比、构图目的、景深和背景虚化程度。

光线与滤镜：
说明光源方向、落点、高光、阴影、肤质、衣料效果、色温、饱和度、对比度和暗部质感。
```

允许在模块解析开头用简短条目说明：

```text
画面主目标
时间切片
轻微主事件
```

但不得只输出这些条目后直接进入最终提示词。

内部逐步推理文本不得原样公开。对用户展示的是整理后的视觉决策、扩写结果和摄影效果说明。

## 4.3 参数锁定结果填写规则

### 完整回显硬规则

标准详细输出必须逐项核对用户输入。用户显式填写的字段必须全部出现在参数锁定结果中，包括但不限于：

```text
写真风格
人物设定
场景方向
服装方向
配色方向
华丽度
气质标签
五官方向
身形方向
身形吸引力强度
线条重点
镜头方向
光线氛围
滤镜效果
画幅比例
尺寸参数
平台用途
补充要求
```

允许增加 Route、Overlay、用途推断或缺省补全，但必须放在原始锁定字段之后并明确标注为附注或补充方向。

### 人物设定

用户明确填写年龄、成年边界、地域或人物身份时，必须原样保留核心含义，不得只拆入“五官方向”或导演式解析。

示例：

```text
人物设定：24 岁成年东方女性
```

### 写真风格

填写用户明确指定或系统路由判断后的风格。

示例：

```text
写真风格：纯欲曲线生活照
```

### 场景方向

必须保留用户原场景，不得替换。

示例：

```text
场景方向：海边步道
```

### 服装方向

必须保留用户原服装，不得改品类、颜色、版型。

示例：

```text
服装方向：雾蓝色短款吊带 + 白色轻薄开衫 + 浅色短裤
```

### 配色方向

用户填写时必须保留；缺失时按 route 或服装主色补全。配色方向必须贯穿服装、发饰、场景、光线和滤镜。

示例：

```text
配色方向：朱红 + 暖金 + 珍珠白，朱红为主色，暖金用于刺绣和腰封，珍珠白用于披帛和珠玉点缀
```

### 华丽度

用户填写时必须保留；缺失时按 route 补全，不适用时写“不特别强调”。华丽度不能只作为标签重复，必须落实为服装层次、刺绣密度、材质、发饰首饰等级与背景复杂度。

示例：

```text
华丽度：高，以大袖衫层次、刺绣腰封、精致步摇和少量珠玉高光呈现，背景华丽但虚化
```

### 气质标签

保留用户气质，并可补充少量相容词。

示例：

```text
气质标签：清纯、纯欲、安静、有吸引力
```

### 五官方向

如果用户未填写，根据 route 默认补全。

示例：

```text
五官方向：清冷淡颜
```

### 身形方向

如果用户未填写，根据 route 默认补全。

示例：

```text
身形方向：自然吸引力曲线
```

### 身形吸引力强度

如用户未填写，纯欲曲线类默认“中”，其他风格可写“不特别强调”。

示例：

```text
身形吸引力强度：中
```

### 线条重点

如用户未填写，按风格补全；如不适用，写“自然比例，不特别强调”。

示例：

```text
线条重点：脖颈、锁骨、腰线、小腹、大腿
```

### 镜头方向

保留用户镜头；缺失时按 route 补全。

示例：

```text
镜头方向：轻侧身站姿，大腿以上构图
```

### 光线氛围

保留用户光线；缺失时按 route 补全。

示例：

```text
光线氛围：傍晚柔和自然光
```

### 滤镜效果

保留用户滤镜；缺失时按 route 补全。

示例：

```text
滤镜效果：冷白纯欲生活照滤镜
```

### 画幅比例

如用户未填写，默认 9:16。

示例：

```text
画幅比例：9:16
```

### 尺寸参数

包括用户明确填写的像素尺寸、分辨率、宽高或清晰度尺寸要求，例如 `1080×1920`、`2048×2048`、`2K`、`4K`。用户填写时必须原值锁定，并写入最终提示词第一句。用户未填写时标注“未指定”或省略，不得虚构具体像素值。

示例：

```text
尺寸参数：1080×1920
```

### 平台用途

如用户未填写，可根据语境补全为“生活照 / 社交媒体 / 小红书种草 / 作品集”。

示例：

```text
平台用途：小红书种草
```

### 补充要求

必须保留用户原补充要求。

示例：

```text
补充要求：服装完整清晰，不要低俗，不要复杂背景
```

## 4.4 最终提示词写法规则

最终提示词不能机械复述字段，而是要根据导演规则进行详细的描述或扩写。

错误写法：

```text
生成一张纯欲曲线生活照，场景是海边步道，服装是雾蓝色短款吊带，气质是清纯、纯欲、安静、有吸引力，镜头是大腿以上构图，光线是傍晚自然光。
```

正确写法：

```text
生成一张 9:16 竖版、写实电影生活剧照感的纯欲曲线女性生活照。傍晚的海边步道上，画面主体是一位 20–28 岁年轻成年东方女性，成年气质明确，具有自然东方女性特征，不过度欧美化，不幼态化，没有未成年感。她在散步中短暂停下，身体轻微侧向镜头，重心自然落在后侧腿上，肩颈放松，眼神安静地看向远处海面。

人物五官方向为清冷淡颜，脸型柔和流畅，眉眼清淡，眼神克制自然，妆容清透干净，不网红脸，不假面感。人物身形为自然吸引力曲线，身形吸引力强度为中。雾蓝色短款吊带、白色轻薄开衫和浅色短裤形成清爽的冷色层次，服装完整自然，海风轻轻带动开衫边缘。柔和侧光让脖颈、锁骨、腰线、小腹和大腿线条自然呈现，整体是成年女性克制、有吸引力的身体线条表达，不低俗、不裸露、不情趣化。

镜头采用大腿以上构图，人物主体清晰，既保留脸部表情和服装层次，也自然呈现腰线与腿部比例。背景只保留远处海面、步道栏杆和傍晚天光，浅景深虚化，不喧宾夺主。整体采用冷白纯欲生活照滤镜，色温偏冷但不过曝，饱和度克制，肤色干净清透，画面真实、柔和、有电影生活剧照感。
```

## 4.5 负面约束写法规则

负面约束必须单独输出。

本文件的最终原则是：

```text
用户要提示词，就给可复制提示词；
用户要参数组合，就给完整可调用参数；
用户要优化，就给优化后的最终版本；
用户要诊断，就指出问题并给修复方案；
用户要安全改写，就保留效果并降低风险；
用户要反推，就分析后输出可用提示词；
用户要 Skill 文档，就一次输出一个完整核心文档。
```

输出格式必须服务于一个目标：

```text
让用户不需要二次整理，就可以直接使用。
```

所有模块必须遵守本文件。
