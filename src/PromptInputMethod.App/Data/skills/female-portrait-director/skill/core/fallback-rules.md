# 女性人像提示词导演 Skill｜参数缺失补全规则

版本编号：`FEMALE-PORTRAIT-DIRECTOR-V1.4`
文档类型：核心规则文档
适用范围：所有女性人像提示词生成、参数组合推荐、提示词优化、审查友好改写、图片反推提示词、风格路由任务
核心职责：当用户输入参数不完整时，按照风格、场景、平台用途和安全边界自动补全缺失参数，确保最终提示词完整、稳定、可复制，同时不覆盖用户已锁定参数

---

## 1. 文档定位

本文件用于规定女性人像提示词导演 Skill 的参数缺失补全规则。

它负责解决以下问题：

1. 用户只输入少量参数时，系统如何自动补全；
2. 用户没有填写五官时，如何根据风格补全；
3. 用户没有填写身形时，如何根据风格补全；
4. 用户没有填写配色方向和华丽度时，如何按 route 补全；
5. 用户没有填写镜头时，如何根据画面目标补全；
6. 用户没有填写光线和滤镜时，如何根据风格补全；
7. 用户没有填写平台用途时，如何推导合理用途；
8. 用户没有填写负面约束时，如何自动继承基础负面约束；
9. 用户输入极简时，如何避免生成机械、空泛、模板化提示词；
10. 补全内容如何不覆盖用户已填写参数。

## 2. 补全总原则

参数缺失时，系统可以补全。

但补全必须遵守以下原则：

```text
1. 用户已填写参数不得被覆盖；
2. 补全内容必须符合当前 route 风格；
3. 补全内容必须符合成年边界与安全表达；
4. 补全内容必须与用户补充要求一致；
5. 补全内容必须服务画面意图；
6. 补全内容必须能被导演式扩写融合；
7. 补全内容不得机械堆叠；
8. 补全内容不得引入新的风格冲突；
9. 补全内容不得让画面变得过度复杂；
10. 补全后最终提示词必须像一张真实可拍摄或可信生成的照片。
```

核心判断：

```text
用户写了什么，锁定什么；
用户没写什么，按风格补全；
补全不是替换；
补全不是扩张到失控；
补全是让画面完整成立。
```

## 3. 缺失参数判断流程

每次生成提示词前，系统必须先检查以下字段是否缺失：

```text
写真风格：
场景方向：
服装方向：
配色方向：
华丽度：
气质标签：
五官方向：
身形方向：
身形吸引力强度：
线条重点：
镜头方向：
光线氛围：
滤镜效果：
画幅比例：
尺寸参数：
平台用途：
补充要求：
```

判断方式：

```text
1. 用户明确填写 → 锁定，不补全为其他内容；
2. 用户未填写但可从语境推导 → 语义补全；
3. 用户未填写且无法推导 → 按 route 默认值补全；
4. 用户未填写且当前任务不需要 → 标记为“不特别强调”；
5. 用户填写内容冲突 → 调用 conflict-resolution.md 处理；
6. 用户填写内容高风险 → 调用 safety-boundary.md 安全转译。
```

## 3.1 缺少配色方向和华丽度时的补全

`配色方向` 和 `华丽度` 是正式字段，但并非每条 route 都需要强强调。

补全规则：

```text
1. 用户明确填写 → 锁定，不得替换；
2. 用户没有填写，但服装颜色清晰 → 从服装提取主色，并按 route 补充克制的辅色或点缀色；
3. 用户没有填写，且 route 定义了默认配色 → 使用 route 默认配色；
4. 用户没有填写华丽度，且 route 明确依赖装饰密度 → 使用 route 默认华丽度；
5. 用户没有填写华丽度，且当前 route 不依赖华丽表达 → 标记为“不特别强调”；
6. 补全后必须在服装、发饰、场景、光线和滤镜中保持一致。
```

古风类默认：

```text
配色控制为 1 个主色 + 1 个辅色 + 1 个金属或珠宝点缀色；
华丽度通过衣料层次、刺绣密度、发饰等级、珠宝高光和背景复杂度落实；
不得只重复“华丽”标签，不得堆满装饰元素。
```

### 5.3 场景补全规则

场景补全必须遵守：

```text
1. 每次只选择一个主场景；
2. 不要把多个场景混合；
3. 不要把场景库里的道具全部塞进提示词；
4. 每张图只保留 2–3 个环境细节；
5. 场景必须服务人物和风格；
6. 电商类场景必须减少干扰；
7. 古风类场景必须虚化弱化，不抢人物；
8. 生活照场景必须真实自然，不像影楼布景。
```

```text
补充要求：服装完整清晰，颜色准确，不要色差，不要复杂背景，不遮挡服装主体
```

## 17 补全自检表

每次补全后必须检查：

```text
1. 补全内容是否覆盖了用户已填写参数？
2. 补全风格是否符合用户意图？
3. 补全场景是否与服装、气质一致？
4. 补全服装是否安全、完整、符合风格？
5. 补全五官是否避免人物趋同？
6. 补全身形是否符合安全边界？
7. 补全镜头是否服务画面目标？
8. 补全光线和滤镜是否冲突？
9. 补全平台用途是否覆盖主风格？
10. 补全负面约束是否包含基础安全边界？
11. 补全后是否能进行导演式扩写？
12. 最终提示词是否像一张真实照片，而不是字段拼接？
```

如任意一项不合格，必须重新补全。

## 18. 最终原则

本文件的最终原则是：

```text
补全是为了让画面完整，不是为了替用户做主；
补全是为了让提示词稳定，不是为了把风格改成默认模板；
补全是为了支持导演式扩写，不是为了机械填满字段。
```

判断补全是否成功的标准：

```text
用户输入的核心内容仍然清晰存在；
缺失参数被合理补足；
风格、场景、服装、五官、身形、镜头、光线、滤镜互相一致；
最终提示词可以自然扩写成一张可信照片；
安全边界没有丢失。
```

所有模块必须遵守本文件。
