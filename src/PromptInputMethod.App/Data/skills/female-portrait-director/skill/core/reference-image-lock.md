## 1. 文档定位

本文件用于锁定上传图片中的人物身份特征和产品核心视觉特征。它不替代 Route、Overlay 或导演模式，而是在风格转换前声明哪些内容不得被扩写覆盖。

本文件必须被 `tools/reference-image-generate.md` 继承。服装试穿任务还必须继承 `routes/commercial/ecommerce-tryon.md`。

## 2. 图片角色

每张上传图片必须先标记一个角色：

| 角色 | 用途 | 默认锁定范围 |
| --- | --- | --- |
| `identity_reference` | 本人或已授权成年人物参考图 | 脸型、眉眼、鼻唇、可识别五官特征 |
| `product_reference` | 服装、配饰、包装或普通实物 | 品类、轮廓、颜色、材质、图案、关键结构 |
| `style_reference` | 仅参考风格、构图、光线或氛围 | 不锁定人物身份或产品主体 |
| `edit_target` | 需要直接修改的原图 | 用户未要求改变的主体和构图关系 |

图片角色不明确时，先询问用户，不自行猜测。

## 3. 锁定优先级

```text
安全边界
> 参考图主体锁定
> 用户明确修改要求
> 用户选择的目标 Route
> Overlay
> Route 默认补全
```

Route 和 Overlay 只能修改未锁定内容，不得用默认五官、默认商品或默认造型覆盖参考图主体。

## 4. 人物身份锁定

### 4.1 默认锁定

`identity_reference` 默认锁定：

```text
脸型与脸部比例；
眉形、眼型和眼距；
鼻型、唇形和嘴角特征；
可观察的稳定五官特征；
人物整体可识别度。
```

默认允许调整：

```text
发型；
妆容；
服装；
姿态；
场景；
镜头；
光线；
滤镜；
目标风格。
```

用户明确要求“整体人物尽量不变”时，再扩大锁定范围到发型、身形、姿态或构图。

### 4.2 授权判断

允许：

```text
用户明确说明是自己的自拍；
用户明确说明人物已经授权；
明确成年人物的正常写真、换装、妆造和场景转换。
```

必须先确认：

```text
上传第三方人物图片，但未说明授权；
人物年龄无法判断，且请求包含性感、纯欲、贴身穿搭或其他性化方向。
```

拒绝：

```text
未成年人性化；
未获授权的真实人物身份复刻；
明星、公众人物或第三方人物的欺骗性冒充；
偷拍、侵犯隐私、骚扰、诽谤；
色情深度伪造或非自愿情境。
```

## 5. 产品锁定

`product_reference` 默认采用核心视觉高保真：

```text
锁定品类；
锁定整体轮廓；
锁定主色和关键配色；
锁定材质；
锁定图案；
锁定关键结构和装饰；
避免产品被遮挡、裁切、变形或被 Route 默认物件替换。
```

Logo、包装文字和小字应尽量保留，但不承诺逐字准确。用户明确要求文字精确时，说明生成模型限制；必要时建议后续精确编辑或合成，不静默承诺。

产品图片默认按用户有权使用处理。不得用于仿冒、虚假宣传或违法用途。

## 6. 多图组合

支持：

```text
一张人物自拍 + 一张服装图；
一张人物自拍 + 一个或多个产品图；
人物图 + 产品图 + 风格参考图；
待编辑图片 + 补充产品图；
单独产品图 + 目标广告风格。
```

多图任务必须先形成内部图片角色表：

```text
图片 1：identity_reference
图片 2：product_reference
图片 3：style_reference
目标 Route：
允许改变：
禁止改变：
```

## 7. 执行规则

```text
1. 识别每张图片角色；
2. 检查人物授权、成年边界和用途；
3. 建立人物 / 产品锁定表；
4. 读取目标 Route 和可选 Overlay；
5. 通过 director-gate 完成内部导演设计；
6. 使用图片编辑、参考图生成或多图合成能力；
7. 检查人物五官或商品主体是否漂移；
8. 漂移明显时进行一次定向重试；
9. 仍无法满足时明确说明限制，不静默退化为普通文本生图。
```

## 8. 与其他工具的边界

- `image-to-prompt`：分析图片并输出提示词，不负责保留真实人物身份。
- `reference-image-generate`：把上传图片作为生成约束，默认直接返回图片。
- `ecommerce-tryon`：服装试穿专项 Route，负责服装还原优先级。

## 9. 输出前检查

```text
- 图片角色是否明确？
- 人物是否为本人或已授权成年人物？
- 人物默认五官是否保持可识别？
- 产品品类、轮廓、颜色、材质、图案和关键结构是否保留？
- Route 和 Overlay 是否只修改未锁定内容？
- 是否避免未成年人性化、欺骗性冒充和侵犯隐私用途？
- 是否避免对 Logo 和小字精确度作不真实承诺？
- 是否在保真失败时明确说明限制？
```
