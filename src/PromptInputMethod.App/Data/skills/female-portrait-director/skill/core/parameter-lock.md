# 女性人像提示词导演 Skill｜参数锁定规则

版本编号：`FEMALE-PORTRAIT-DIRECTOR-V1.4.1`
文档类型：核心规则文档
适用范围：所有 routes、overlays、tools、examples、tests 中涉及用户参数解析、提示词生成、提示词优化、参数组合推荐、审查友好改写、图片反推提示词的任务
核心职责：确保用户明确输入的参数不被替换、不被弱化、不被擅自优化为其他方向

---

## 1. 文档定位

本文件是女性人像提示词导演 Skill 的参数锁定规则。

参数锁定是整个 Skill 的最高优先级规则之一。

本文件负责规定：

1. 哪些内容属于用户锁定参数；
2. 系统如何识别用户显式参数；
3. 系统可以怎样扩写用户参数；
4. 系统不允许怎样改写用户参数；
5. 当参数冲突时如何处理；
6. 当参数缺失时如何补全；
7. 当用户要求优化时如何保留原参数；
8. 当用户要求审查友好改写时如何保留原画面目标；
9. 当 route、overlay、tool 同时作用时，如何保证参数不被覆盖。

本文件必须被以下所有模块继承：

```text
skill/skill.md
skill/style-registry.md
skill/references/director-expansion.md
skill/references/visual-libraries.md

skill/overlays/

```

## 2. 最高规则

用户明确填写的参数拥有最高优先级。

系统只能在用户参数基础上进行：

```text
扩写
细化
稳定化
摄影化
导演化
合规化
```

系统不得擅自进行：

```text
替换
弱化
转移
忽略
反向改写
改成更常见但不是用户要求的方向
改成系统认为更容易出图但违背用户输入的方向
```

核心原则：

```text
用户写了什么，就必须保留什么；
用户没写什么，系统才可以补全；
用户写得简单，可以扩写；
用户写得具体，不得替换；
用户写得冲突，优先保留明确参数，再通过导演式规则降低冲突；
用户写得敏感，保留画面目标，但必须转化为成年、时尚、克制、非低俗表达。
```



### 3.14 补充要求

---

### 4.3 第三步：区分“锁定参数”和“待补全参数”

锁定参数：

```text

```

待补全参数：

```text
用户没有写，但生成完整提示词必须需要的内容。
```

规则：

```text

待补全参数可以根据风格默认值补全；
补全内容不得与锁定参数冲突。
```

### 4.4 第四步：执行扩写而非替换

扩写的定义：

```text
在保留原参数的基础上增加具体摄影细节、材质细节、动作链、光线落点和画面逻辑。
```

例如：

用户输入：

```text

```

扩写：

```text
白色或米白色针织开衫，材质柔软，领口自然敞开，衣摆随坐姿轻微堆叠，呈现温柔生活感。
```

不是扩写：

```text
白色连衣裙。
```

## 5. 参数冲突时的锁定规则

当参数冲突时，不直接替换用户参数，而是优先保留用户明确内容，再降低冲突。

---

### 5.1 清冷风格 + 红金配色

用户输入：


```text

```

错误处理：

```text

```

## 8. 审查友好改写中的参数锁定

高风险词替换规则：

```text

暴露 → 轻盈穿搭 / 露肤克制 / 服装完整

```

示例：

用户输入：

```text

```

安全改写：

```text
成年女性冷白纯欲生活照，海边自然场景，贴身吊带作为生活化时尚穿搭出现，服装完整，通过肩颈、锁骨、腰线和腿部比例表现克制的成年女性线条美，整体真实、自然、有吸引力但不低俗。
```

不得改成：

```text

```

因为这会删除用户原本的身形线条目标。

## 10. Route 调用中的参数锁定

Route 只能提供风格扩写规则，不得覆盖锁定参数。

例如：

`routes/lifestyle/french-lazy.md` 可能默认服装为：

```text

```

但用户输入：

```text
服装方向：黑色针织背心 + 牛仔裤
```

则 route 必须保留用户服装，并将其法式化扩写：

```text
黑色针织背心作为简洁法式私服出现，搭配牛仔裤，整体松弛自然，保留法式窗边生活感。
```

不得改成：

```text
白衬衫 + 米白半裙。
```

```text
构图更稳定
适度留白
降低背景干扰
主体更明确
```

不得调整：

```text
改变用户风格
改变画幅
把人物变得太小
删除用户重点服装或场景
```

## 12. 图片反推提示词中的参数锁定

图片反推时，用户可能要求：

```text
保留原图风格
改成 3D CG
改成古风
保留构图
去掉某个元素
替换某个道具
```

规则：

```text
用户要求保留的必须保留；
用户要求改变的必须改变；
用户未要求改变的核心视觉尽量保留；
不得机械复制参考图；
不得与安全边界冲突。
```

示例：

用户输入：

```text
推导这张图的 AI 提示词，将风格改成 3D CG 写实风，将人物手中的报纸改成手机。
```

必须锁定：

```text
原图构图
人物姿态
整体氛围
3D CG 写实风
报纸替换为手机
```

不得输出：

```text
仍然拿着报纸；
改成真人摄影；
完全改变构图。
```

## 12.1 参考图保留直接生成

用户上传图片并要求保留人物身份或产品主体直接生成时，读取 [reference-image-lock.md](reference-image-lock.md)。

内部必须先标记：

```text
identity_reference：本人或已授权成年人物图片
product_reference：服装、配饰、包装或普通实物
style_reference：仅参考风格、构图、光线或氛围
edit_target：需要直接修改的原图
```

锁定优先级：

```text
安全边界
> 参考图主体锁定
> 用户明确修改要求
> 用户选择的目标 Route
> Overlay
> Route 默认补全
```

参考图保留生成与图片反推不同：前者允许在授权范围内锁定人物身份并直接生成图片；后者只转译可观察视觉规律，不复制真实人物身份。

## 14. 参数锁定自检表

每次生成最终提示词前，必须检查以下问题。

```text
1. 用户填写的写真风格是否被替换？
2. 用户填写的场景是否被替换？
3. 用户填写的服装是否被替换？
4. 用户填写的配色方向是否被替换、弱化或漏写？
5. 用户填写的华丽度是否被替换，是否已经落实为可见设计？
6. 用户填写的气质是否被替换？
7. 用户填写的五官是否被替换？
8. 用户填写的身形是否被替换？
9. 用户填写的线条重点是否被删除？
10. 用户填写的镜头是否被替换？
11. 用户填写的光线是否被替换？
12. 用户填写的滤镜是否被替换？
13. 用户填写的画幅是否被替换或从最终提示词第一句中遗漏？
14. 用户填写的尺寸参数是否被替换、虚构或从最终提示词第一句中遗漏？
15. 用户填写的平台用途是否被错误覆盖？
16. 用户填写的补充要求是否被忽略？
17. route 默认值是否覆盖了用户输入？
18. overlay 是否替代了主风格？
19. tool 优化是否改变了原意？
20. 审查友好改写是否过度保守，导致画面目标丢失？
```

如果任何一项答案为“是”，必须修正后再输出。

## 15. 常见错误示例

---

### 15.1 错误：为了更好看改风格

用户输入：

```text
写真风格：清纯生活照
场景：咖啡馆
```

错误输出：

```text
高级杂志大片风咖啡馆写真。
```

正确：

```text
保持清纯生活照，只增强咖啡馆真实生活感、自然光和温柔气质。
```

---

### 15.2 错误：为了稳定改服装

用户输入：

```text
服装：雾蓝色短款吊带 + 白色轻薄开衫 + 浅色短裤
```

错误输出：

```text
白色宽松衬衫 + 牛仔裤。
```

正确：

```text
保留雾蓝色短款吊带、白色轻薄开衫和浅色短裤，只通过完整服装、自然姿态和冷白光线增强稳定性。
```

---

### 15.3 错误：为了安全删掉身形目标

用户输入：

```text
线条重点：锁骨、腰线、小腹、大腿
```

错误输出：

```text
半身头像，穿宽松衣服。
```

正确：

```text
保留线条重点，但通过服装完整、轻侧身、柔和光线、大腿以上构图表现成年女性自然线条美。
```

---

### 15.4 错误：平台用途覆盖主风格

用户输入：

```text
写真风格：复古港风写真
平台用途：小红书
```

错误输出：

```text
普通小红书清新穿搭图。
```

正确：

```text
保持复古港风写真，只增强主体清晰、构图稳定和社交平台传播感。
```

---

### 15.5 错误：Overlay 替换 Route

用户输入：

```text
写真风格：日系清透写真
气质：冷艳御姐
```

错误输出：

```text
夜色霓虹冷艳都市写真。
```

正确：

```text
保留日系清透写真，只在人物表情、眼神和穿搭气场上加入冷艳御姐气质。
```

## 16. 最终原则

参数锁定的最终原则是：

```text
用户参数是边界；
route 是风格执行器；
overlay 是气质增强器；
tool 是功能处理器；
references 是扩写工具箱；
core 是底层规则；
任何模块都不得覆盖用户明确参数。
```

本文件的核心判断标准是：

```text
如果用户看到最终提示词后，觉得“这还是我刚才输入的画面，只是被扩写得更完整、更稳定、更像真实照片”，则参数锁定成功。

如果用户觉得“你把我的画面改成了另一个东西”，则参数锁定失败。
```

所有模块必须遵守本文件。
